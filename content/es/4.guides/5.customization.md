Esta guía te muestra cómo personalizar varios aspectos del Auth Starter para que coincidan con las necesidades de tu aplicación.

## Personalización de Plantillas de Email

El starter usa plantillas Thymeleaf para renderizado de emails (solo proveedor SMTP). Los usuarios de Brevo gestionan plantillas en el panel de Brevo.

### Ubicaciones de Plantillas

Coloca tus plantillas personalizadas en:
```
src/main/resources/templates/
├── verification.html        # Plantilla de verificación de email
└── password-reset.html      # Plantilla de restablecimiento de contraseña
```

### Variables de Plantilla Disponibles

**Email de Verificación (`verification.html`):**

| Variable | Tipo | Descripción | Ejemplo |
|----------|------|-------------|---------|
| `${name}` | String | Nombre del usuario | "John" |
| `${confirmationUrl}` | String | Enlace de verificación de email | "https://app.com/verify-email?code=...&email=..." |
| `${expiration}` | Integer | Tiempo hasta expiración | 24 |
| `${unit}` | String | Unidad de tiempo (configurable) | "hours" o "minutes" |
| `${currentYear}` | Integer | Año actual para pie de página | 2026 |

**Email de Restablecimiento de Contraseña (`password-reset.html`):**

| Variable | Tipo | Descripción | Ejemplo |
|----------|------|-------------|---------|
| `${name}` | String | Nombre del usuario | "John" |
| `${resetUrl}` | String | Enlace de restablecimiento de contraseña | "https://app.com/reset-password?code=...&email=..." |
| `${expiration}` | Integer | Tiempo hasta expiración | 24 |
| `${unit}` | String | Unidad de tiempo (configurable) | "hours" o "minutes" |
| `${currentYear}` | Integer | Año actual para pie de página | 2026 |


### Ejemplo: Email de Verificación Personalizado Mínimo

Crea tu propia plantilla en `src/main/resources/templates/verification.html`:

```html
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <meta charset="UTF-8">
    <title>Verifica tu Email</title>
</head>
<body>
    <h1>¡Bienvenido, <span th:text="${name}">Usuario</span>!</h1>

    <p>Haz clic en el enlace de abajo para verificar tu email:</p>
    <a th:href="${confirmationUrl}">Verificar Email</a>

    <p>Este enlace expira en <span th:text="${expiration}">24</span>
       <span th:text="${unit}">horas</span>.</p>

    <footer>
        <p>&copy; <span th:text="${currentYear}">2026</span> TuApp</p>
    </footer>
</body>
</html>
```

::tip{title="Consejos de Estilo"}
El starter incluye plantillas con estilo profesional por defecto. Úsalas como referencia al crear tus plantillas personalizadas. Están ubicadas en el JAR del starter bajo `templates/`.
::

## Personalizando Unidades de Tiempo

Cambia cómo aparecen las duraciones de tiempo en las notificaciones por email.

| Propiedad | Descripción | Predeterminado |
| :--- | :--- | :--- |
| `ctlabs.auth.verification.unit-minutes` | Etiqueta para minutos | `"minutes"` |
| `ctlabs.auth.verification.unit-hours` | Etiqueta para horas | `"hours"` |

**Ejemplo - Etiquetas en español:**

::code-group
```properties [application.properties]
ctlabs.auth.verification.unit-minutes=minutos
ctlabs.auth.verification.unit-hours=horas
```

```yaml [application.yml]
ctlabs:
  auth:
    verification:
      unit-minutes: minutos
      unit-hours: horas
```
::

## Validación de Contraseñas

Personaliza los requisitos de contraseña usando patrones regex.

| Propiedad | Descripción | Predeterminado |
| :--- | :--- | :--- |
| `ctlabs.auth.password.validation-regex` | Patrón regex para validación | `^(?=.*[0-9])(?=.*[a-z])(?=.*[A-Z])(?=\S+$).{8,20}$` |
| `ctlabs.auth.password.validation-message` | Mensaje de error mostrado a usuarios | (Ver abajo) |

**Requisitos predeterminados:**
- 8-20 caracteres
- Al menos un dígito
- Al menos una letra minúscula
- Al menos una letra mayúscula
- Sin espacios en blanco

**Ejemplo - Requiere caracteres especiales:**

::code-group
```properties [application.properties]
# Requiere 12+ caracteres, 1 dígito, 1 carácter especial
ctlabs.auth.password.validation-regex=^(?=.*[0-9])(?=.*[!@#$%^&*])(?=\\S+$).{12,}$
ctlabs.auth.password.validation-message=La contraseña debe tener al menos 12 caracteres con un número y un carácter especial
```

```yaml [application.yml]
ctlabs:
  auth:
    password:
      validation-regex: "^(?=.*[0-9])(?=.*[!@#$%^&*])(?=\\S+$).{12,}$"
      validation-message: La contraseña debe tener al menos 12 caracteres con un número y un carácter especial
```
::

::warning{title="Escape de Regex"}
En archivos `.properties`, las barras invertidas deben escaparse: `\\S` en lugar de `\S`. En archivos YAML, usa comillas y barras invertidas simples.
::

## Expiración de Tokens

Ajusta cuánto tiempo permanecen válidos los tokens y códigos de verificación.

| Propiedad | Descripción | Predeterminado |
| :--- | :--- | :--- |
| `ctlabs.auth.jwt.expiration` | Vida útil del token de acceso | `15m` |
| `ctlabs.auth.refresh-token.expiration` | Vida útil del token de renovación | `7d` |
| `ctlabs.auth.verification.email-link-expiration` | Validez del enlace de verificación de email | `24h` |
| `ctlabs.auth.verification.phone-code-expiration` | Validez del código de verificación por teléfono | `10m` |

**Formatos soportados:**
- Simple: `15m`, `24h`, `7d`
- ISO-8601: `PT15M`, `PT24H`, `P7D`

**Ejemplo - Sesiones extendidas:**

::code-group
```properties [application.properties]
# Tokens de acceso de 30 minutos
ctlabs.auth.jwt.expiration=30m

# Tokens de renovación de 30 días
ctlabs.auth.refresh-token.expiration=30d

# Verificación de email de 48 horas
ctlabs.auth.verification.email-link-expiration=48h

# Códigos de teléfono de 15 minutos
ctlabs.auth.verification.phone-code-expiration=15m
```

```yaml [application.yml]
ctlabs:
  auth:
    jwt:
      expiration: 30m
    refresh-token:
      expiration: 30d
    verification:
      email-link-expiration: 48h
      phone-code-expiration: 15m
```
::

::warning{title="Consideraciones de Seguridad"}
- **Tokens de acceso**: Más corto es más seguro (se recomiendan 10-30 minutos)
- **Tokens de renovación**: Balancea seguridad con experiencia de usuario (7-30 días)
- **Códigos de verificación**: Mantenlos cortos por seguridad (5-15 minutos para teléfono, 24-48 horas para email)
::
