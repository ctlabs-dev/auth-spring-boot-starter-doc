Esta guía te lleva paso a paso por la configuración de **verificación real por email** para registros de usuarios. Al final, los usuarios recibirán emails de verificación y deberán hacer clic en un enlace para activar sus cuentas.

::note
El [Inicio Rápido](/getting-started/quickstart) usa `Provider.NONE` para auto-verificación. Esta guía muestra cómo habilitar **verificación real por email** con prueba de propiedad.
::

## Requisitos Previos

Elige un proveedor de email:

### Opción A: SMTP (Gmail, Outlook, etc.)
- Una cuenta de Gmail (u otro proveedor SMTP)
- Contraseña de aplicación habilitada (para Gmail: [Guía de Contraseñas de App](https://support.google.com/accounts/answer/185833))

### Opción B: Brevo (Recomendado)
- [Cuenta gratuita de Brevo](https://www.brevo.com/) (300 emails/día)
- Clave API del [Panel de Brevo](https://app.brevo.com/settings/keys/api)
- Dos plantillas de email creadas (verificación + restablecimiento de contraseña)

---

## Paso 1: Configurar tu Proveedor

### Usando SMTP (Ejemplo con Gmail)

::code-group
```properties [application.properties]
# Configuración del Proveedor de Email
ctlabs.auth.notifications.mail.provider=SMTP
ctlabs.auth.notifications.mail.frontend-url=http://localhost:3000

# Ajustes SMTP
ctlabs.auth.notifications.mail.smtp.from-email=tu-email@gmail.com
ctlabs.auth.notifications.mail.smtp.from-name=Nombre de tu App

# Configuración SMTP de Spring Mail
spring.mail.host=smtp.gmail.com
spring.mail.port=587
spring.mail.username=tu-email@gmail.com
spring.mail.password=tu-contraseña-de-app
spring.mail.properties.mail.smtp.auth=true
spring.mail.properties.mail.smtp.starttls.enable=true
```

```yaml [application.yml]
ctlabs:
  auth:
    notifications:
      mail:
        provider: SMTP
        frontend-url: http://localhost:3000
        smtp:
          from-email: tu-email@gmail.com
          from-name: Nombre de tu App

spring:
  mail:
    host: smtp.gmail.com
    port: 587
    username: tu-email@gmail.com
    password: tu-contraseña-de-app
    properties:
      mail:
        smtp:
          auth: true
          starttls:
            enable: true
```
::

### Usando Brevo

Primero, crea tus plantillas en Brevo:

1. Ve a [Plantillas de Brevo](https://app.brevo.com/camp/lists/template)
2. Crea dos plantillas:
   - **Plantilla de Verificación de Email** - Anota el ID de Plantilla (ej., `1`)
   - **Plantilla de Restablecimiento de Contraseña** - Anota el ID de Plantilla (ej., `2`)

**Variables de plantilla requeridas:**
- `{{params.name}}` - Nombre del usuario
- `{{params.confirmationUrl}}` - Enlace de verificación (incluye parámetros `code` y `email`) - **solo para verificación de email**
- `{{params.resetUrl}}` - Enlace de restablecimiento de contraseña (incluye parámetros `code` y `email`) - **solo para restablecimiento de contraseña**
- `{{params.expiration}}` - Tiempo de expiración (número)
- `{{params.unit}}` - Unidad de tiempo (ej., "horas")
- `{{params.currentYear}}` - Año actual

::note
La URL de verificación estará en el formato: `http://tu-frontend.com/verify-email?code=abc123&email=usuario@example.com`

La URL de restablecimiento de contraseña estará en el formato: `http://tu-frontend.com/reset-password?code=abc123&email=usuario@example.com`
::

::code-group
```properties [application.properties]
# Configuración del Proveedor de Email
ctlabs.auth.notifications.mail.provider=BREVO
ctlabs.auth.notifications.mail.frontend-url=http://localhost:3000

# Ajustes de Brevo
ctlabs.auth.notifications.mail.brevo.api-key=xkeysib-tu-clave-api-aqui
ctlabs.auth.notifications.mail.brevo.verification-template-id=1
ctlabs.auth.notifications.mail.brevo.password-reset-template-id=2
```

```yaml [application.yml]
ctlabs:
  auth:
    notifications:
      mail:
        provider: BREVO
        frontend-url: http://localhost:3000
        brevo:
          api-key: xkeysib-tu-clave-api-aqui
          verification-template-id: 1
          password-reset-template-id: 2
```
::

## Paso 2: Crear Plantillas de Email (Solo SMTP)

Si usas SMTP, crea plantillas Thymeleaf en tu proyecto:

**Archivo:** `src/main/resources/templates/verification.html`

```html
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <title>Verifica tu dirección de email</title>
</head>
<body>
    <h1>¡Bienvenido, <span th:text="${name}">Usuario</span>!</h1>
    <p>Gracias por registrarte. Por favor verifica tu dirección de email:</p>
    <a th:href="${confirmationUrl}">Verificar Email</a>
    <p>Este enlace expira en <strong th:text="${expiration}">24</strong> <strong th:text="${unit}">horas</strong>.</p>
</body>
</html>
```

**Archivo:** `src/main/resources/templates/password-reset.html`

```html
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <title>Restablece tu contraseña</title>
</head>
<body>
    <h1>Solicitud de Restablecimiento de Contraseña</h1>
    <p>¡Hola, <span th:text="${name}">Usuario</span>!</p>
    <p>Haz clic en el enlace de abajo para restablecer tu contraseña:</p>
    <a th:href="${resetUrl}">Restablecer Contraseña</a>
    <p>Este enlace expira en <strong th:text="${expiration}">24</strong> <strong th:text="${unit}">horas</strong>.</p>
</body>
</html>
```

::note
Los usuarios de Brevo omiten este paso - las plantillas se gestionan en el panel de Brevo.

La variable `confirmationUrl` contendrá la URL completa con parámetros de consulta `code` y `email` (ej., `http://localhost:3000/verify-email?code=abc123&email=usuario@example.com`).
::

## Paso 3: Probar el Flujo

### 1. Registrar un Nuevo Usuario

```bash
curl -X POST http://localhost:8080/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "nuevousuario@example.com",
    "password": "SecurePass123",
    "firstName": "Jane",
    "lastName": "Doe"
  }'
```

**Respuesta esperada:**
```json
{
  "message": "Registration successful. Please check your email to verify your account."
}
```

### 2. Revisar tu Email

- **SMTP:** Revisa la bandeja de entrada del email registrado
- **Brevo:** Revisa panel de Brevo → Registros → Transaccional para ver el estado de entrega

Deberías recibir un email con un enlace de verificación como:
```
http://localhost:3000/verify-email?code=abc123xyz...&email=usuario@example.com
```

### 3. Verificar la Cuenta

El usuario hace clic en el enlace, que debería llamar a tu backend:

```bash
curl -X POST http://localhost:8080/api/auth/verify-email \
  -H "Content-Type: application/json" \
  -d '{
    "email": "usuario@example.com",
    "code": "abc123xyz..."
  }'
```

**Respuesta esperada:**
```json
{
  "message": "Email verified successfully. You can now log in."
}
```

### 3.1 Reenviar Código (Opcional)

Si el usuario no recibe el email o el código expira, puede solicitar un nuevo código de verificación:

```bash
curl -X POST http://localhost:8080/api/auth/resend-verification \
  -H "Content-Type: application/json" \
  -d '{
    "username": "nuevousuario@example.com",
    "channel": "email"
  }'
```

**Respuesta esperada:**
```json
{
  "message": "A new verification code has been sent to your email."
}
```

::note
Solicitar un nuevo código invalidará cualquier código de verificación previo para ese usuario y canal.
::

### 4. Iniciar Sesión

Ahora el usuario puede iniciar sesión:

```bash
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "nuevousuario@example.com",
    "password": "SecurePass123"
  }'
```

**Respuesta esperada:**
```json
{
  "accessToken": "eyJhbGc...",
  "refreshToken": "5123e231-9bf7..."
}
```

## Solución de Problemas

### Email No Recibido (SMTP)

**Revisa la carpeta de spam** - Los emails de verificación a menudo terminan en spam inicialmente.

**Verifica credenciales:**
```bash
# Probar conexión SMTP
telnet smtp.gmail.com 587
```

**Habilita "Acceso de apps menos seguras"** (Gmail):
- Para Gmail, **debes** usar una [Contraseña de App](https://support.google.com/accounts/answer/185833)
- Las contraseñas regulares no funcionarán con SMTP

**Revisa logs:**
```bash
# Busca errores de mail en los logs de tu app
tail -f logs/application.log | grep -i "mail"
```

### Email No Recibido (Brevo)

**Revisa logs de Brevo:**
1. Ve a [Logs Transaccionales de Brevo](https://app.brevo.com/log/transactional)
2. Busca tu email - el estado debería ser "Entregado"

**Problemas comunes:**
- Clave API inválida → Revisa `ctlabs.auth.notifications.mail.brevo.api-key`
- ID de plantilla no coincide → Verifica IDs de plantilla en configuración
- Email bloqueado → Verifica que el destinatario no esté en lista de bloqueo

### Token Expirado

**Ajusta el tiempo de expiración:**

::code-group
```properties [application.properties]
# Aumentar a 48 horas
ctlabs.auth.verification.email-link-expiration=48h
```

```yaml [application.yml]
ctlabs:
  auth:
    verification:
      email-link-expiration: 48h
```
::

### URL de Frontend Incorrecta

El enlace de verificación usa `ctlabs.auth.notifications.mail.frontend-url` para construir URLs:

```
{frontendUrl}/verify-email?code={code}&email={email}
```

Asegúrate de que esto coincida con la URL real de tu frontend:
- **Desarrollo:** `http://localhost:3000`
- **Producción:** `https://tuapp.com`
