Esta guía te lleva paso a paso por la configuración de **verificación por teléfono** para registros de usuarios. Los usuarios recibirán un código de verificación vía SMS o WhatsApp y deberán ingresarlo para activar sus cuentas.

::note
El [Inicio Rápido](/getting-started/quickstart) usa `Provider.NONE` para auto-verificación. Esta guía muestra cómo habilitar **verificación real por teléfono** con SMS o WhatsApp.
::

## Requisitos Previos

Elige un proveedor:

### Opción A: Twilio (Más Popular)
- [Cuenta de Twilio](https://www.twilio.com/try-twilio) (prueba gratuita con créditos de prueba)
- Número de teléfono de [Consola de Twilio](https://console.twilio.com/us1/develop/phone-numbers/manage/incoming)
- Account SID y Auth Token de [Consola de Twilio](https://console.twilio.com/)
- Para WhatsApp: [Número habilitado para WhatsApp](https://console.twilio.com/us1/develop/sms/try-it-out/whatsapp-learn)

### Opción B: Brevo
- [Cuenta de Brevo](https://www.brevo.com/) (precios de SMS varían por país)
- Clave API del [Panel de Brevo](https://app.brevo.com/settings/keys/api)
- Créditos SMS comprados

---

## Paso 1: Configurar tu Proveedor

### Usando Twilio

::code-group
```properties [application.properties]
# Configuración del Proveedor de Teléfono
ctlabs.auth.notifications.phone.provider=TWILIO
ctlabs.auth.notifications.phone.channel=SMS
ctlabs.auth.notifications.phone.from-phone-number=+1234567890

# Ajustes de Twilio
ctlabs.auth.notifications.phone.twilio.account-sid=ACxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
ctlabs.auth.notifications.phone.twilio.auth-token=tu_auth_token_aqui
```

```yaml [application.yml]
ctlabs:
  auth:
    notifications:
      phone:
        provider: TWILIO
        channel: SMS
        from-phone-number: +1234567890
        twilio:
          account-sid: ACxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
          auth-token: tu_auth_token_aqui
```
::

**Para WhatsApp**, cambia el canal y usa formato WhatsApp:

::code-group
```properties [application.properties]
ctlabs.auth.notifications.phone.channel=WHATSAPP
ctlabs.auth.notifications.phone.from-phone-number=whatsapp:+1234567890
```

```yaml [application.yml]
ctlabs:
  auth:
    notifications:
      phone:
        channel: WHATSAPP
        from-phone-number: whatsapp:+1234567890
```
::

### Usando Brevo

::code-group
```properties [application.properties]
# Configuración del Proveedor de Teléfono
ctlabs.auth.notifications.phone.provider=BREVO
ctlabs.auth.notifications.phone.channel=SMS

# Ajustes de Brevo
ctlabs.auth.notifications.phone.brevo.api-key=xkeysib-tu-clave-api-aqui
ctlabs.auth.notifications.phone.brevo.sender-name=TuApp
```

```yaml [application.yml]
ctlabs:
  auth:
    notifications:
      phone:
        provider: BREVO
        channel: SMS
        brevo:
          api-key: xkeysib-tu-clave-api-aqui
          sender-name: TuApp
```
::

::note
**Brevo solo soporta SMS**, no WhatsApp. El nombre del remitente debe ser máximo 11 caracteres alfanuméricos.
::

## Paso 2: Probar el Flujo

### 1. Registrar un Nuevo Usuario con Teléfono

```bash
curl -X POST http://localhost:8080/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "phoneNumber": "+1234567890",
    "password": "SecurePass123",
    "firstName": "John",
    "lastName": "Doe"
  }'
```

**Respuesta esperada:**
```json
{
  "message": "Registration successful. A verification code has been sent to your phone."
}
```

### 2. Revisar tu Teléfono

Deberías recibir un mensaje SMS o WhatsApp con un código de 6 dígitos:

```
Your verification code is: 123456
This code expires in 10 minutes.
```

### 3. Verificar la Cuenta

El usuario ingresa el código en tu app, que llama a:

```bash
curl -X POST http://localhost:8080/api/auth/verify-phone \
  -H "Content-Type: application/json" \
  -d '{
    "phoneNumber": "+1234567890",
    "code": "123456"
  }'
```

**Respuesta esperada:**
```json
{
  "message": "Phone verified successfully. You can now log in."
}
```

### 4. Iniciar Sesión

Ahora el usuario puede iniciar sesión:

```bash
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "+1234567890",
    "password": "SecurePass123"
  }'
```

**Respuesta esperada:**
```json
{
  "accessToken": "eyJhbGc...",
  "refreshToken": "5123e231-9bf7..."
}
```

---

## Paso 3: Reenviar Código (Opcional)

Si el usuario no recibe el código, puede solicitar uno nuevo:

```bash
curl -X POST http://localhost:8080/api/auth/resend-verification \
  -H "Content-Type: application/json" \
  -d '{
    "username": "+1234567890",
    "channel": "phone"
  }'
```

**Respuesta esperada:**
```json
{
  "message": "A new verification code has been sent to your phone."
}
```

## Solución de Problemas

### SMS No Recibido (Twilio)

**Verifica el formato del número de teléfono:**
- Debe incluir código de país: `+1234567890` (no `234567890`)
- Sin espacios ni guiones

**Verifica credenciales de Twilio:**
```bash
# Probar API de Twilio
curl -X POST "https://api.twilio.com/2010-04-01/Accounts/TU_ACCOUNT_SID/Messages.json" \
  --data-urlencode "Body=Mensaje de prueba" \
  --data-urlencode "From=+1234567890" \
  --data-urlencode "To=+1987654321" \
  -u TU_ACCOUNT_SID:TU_AUTH_TOKEN
```

**Revisa logs de Twilio:**
1. Ve a [Consola de Twilio](https://console.twilio.com/us1/monitor/logs/sms)
2. Busca tu mensaje - el estado debería ser "Entregado"

**Problemas comunes:**
- **Número no verificado** (cuenta de prueba) → Agrega destinatario a [IDs de Llamante Verificados](https://console.twilio.com/us1/develop/phone-numbers/manage/verified)
- **Créditos insuficientes** → Agrega fondos a tu cuenta
- **Restricciones de país** → Algunos países requieren registro

### SMS No Recibido (Brevo)

**Revisa logs de Brevo:**
1. Ve a [Logs SMS de Brevo](https://app.brevo.com/log/sms)
2. Busca tu mensaje

**Problemas comunes:**
- Clave API inválida → Revisa `ctlabs.auth.notifications.phone.brevo.api-key`
- Créditos SMS insuficientes → Compra créditos en el panel de Brevo
- Nombre de remitente inválido → Máximo 11 caracteres alfanuméricos

### WhatsApp No Funciona (Twilio)

**Habilita WhatsApp:**
1. Ve a [Sandbox de WhatsApp de Twilio](https://console.twilio.com/us1/develop/sms/try-it-out/whatsapp-learn)
2. Envía el código de unión desde tu teléfono para activar
3. Usa el formato `whatsapp:+1234567890` para el número de teléfono

**WhatsApp en Producción:**
- Requiere [aprobación de Facebook Business Manager](https://www.twilio.com/docs/whatsapp/tutorial/connect-number-business-profile)
- Puede tomar varios días para aprobación

### Código Expirado

**Ajusta el tiempo de expiración:**

::code-group
```properties [application.properties]
# Aumentar a 15 minutos
ctlabs.auth.verification.phone-code-expiration=15m
```

```yaml [application.yml]
ctlabs:
  auth:
    verification:
      phone-code-expiration: 15m
```
::

## Pruebas en Desarrollo

### Usando Números de Prueba de Twilio Verify

Para probar sin enviar SMS reales:

**Números de teléfono de prueba (sandbox de Twilio):**
- `+15005550006` - Siempre tiene éxito
- `+15005550001` - Siempre falla (número inválido)

**Actualiza tu prueba:**
```bash
curl -X POST http://localhost:8080/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "phoneNumber": "+15005550006",
    "password": "TestPass123",
    "firstName": "Test",
    "lastName": "User"
  }'
```

### Proveedor Mock para Pruebas Locales

Para pruebas unitarias/de integración, cambia de vuelta a `Provider.NONE`:

::code-group
```properties [application-test.properties]
ctlabs.auth.notifications.phone.provider=NONE
```

```yaml [application-test.yml]
ctlabs:
  auth:
    notifications:
      phone:
        provider: NONE
```

## Consideraciones de Costo

### Precios de Twilio (a partir de 2026)
- **SMS:** ~$0.0075 por mensaje (varía por país)
- **WhatsApp:** ~$0.005-0.01 por mensaje
- **Prueba gratuita:** $15 de crédito (suficiente para ~2000 SMS)

### Precios de Brevo
- **Pago por uso:** Varía por país
- **Ejemplo:** ~$0.035 por SMS en EE.UU.
- **Sin nivel gratuito** para SMS

### Consejos de Optimización
- **Usa email para cuentas no críticas** (gratis)
- **Usa WhatsApp** cuando sea posible (más barato que SMS con Twilio)
- **Monitorea el uso** para evitar costos inesperados
