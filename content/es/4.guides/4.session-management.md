Esta guía te muestra cómo usar la funcionalidad de gestión de sesiones proporcionada por el starter.

::note
Entender [Conceptos de Tokens](/concepts/tokens) te ayudará a comprender cómo funcionan las sesiones con JWT y tokens de renovación.
::

---

## Resumen

El Auth Starter usa **tokens de renovación** para gestionar sesiones. Cada vez que un usuario inicia sesión:
- Se genera un nuevo **token de acceso** (JWT) (expira en 15 minutos por defecto)
- Se almacena un nuevo **token de renovación** en la base de datos (expira en 30 días por defecto)

**Funcionalidad disponible:**
- ✅ Ver sesiones activas con detalles (dispositivo, IP, marcas de tiempo)
- ✅ Revocar una sesión específica (cerrar sesión desde un dispositivo)
- ✅ Endpoint de cierre de sesión proporcionado por el starter

---

## Métodos Disponibles

El `AuthService` proporciona estos métodos para gestión de sesiones:

### 1. Obtener Sesiones Activas

```java
List<SessionInfo> getActiveSessions(UUID userId)
```

Devuelve información detallada sobre todas las sesiones activas de un usuario.

**SessionInfo incluye:**
- `UUID id` - Identificador de sesión
- `String deviceInfo` - Información de dispositivo/navegador
- `String ipAddress` - Dirección IP de la sesión
- `Instant createdAt` - Cuándo se creó la sesión
- `Instant expiresAt` - Cuándo expira la sesión

### 2. Revocar Sesión

```java
void revokeSession(UUID sessionId, UUID userId)
```

Revoca una sesión específica por su ID.

## Ejemplo de Implementación

Crea un controlador para exponer la gestión de sesiones a usuarios autenticados:

```java
package com.yourapp.controller;

import dev.ctlabs.starter.auth.application.dto.SessionInfo;
import dev.ctlabs.starter.auth.application.service.AuthService;
import dev.ctlabs.starter.auth.domain.repository.UserRepository;
import org.springframework.security.access.prepost.PreAuthorize;
import org.springframework.security.core.annotation.AuthenticationPrincipal;
import org.springframework.security.core.userdetails.UserDetails;
import org.springframework.web.bind.annotation.*;

import java.util.List;
import java.util.UUID;

@RestController
@RequestMapping("/api/me/sessions")
@PreAuthorize("isAuthenticated()")
public class UserSessionController {

    private final AuthService authService;
    private final UserRepository userRepository;

    public UserSessionController(AuthService authService, UserRepository userRepository) {
        this.authService = authService;
        this.userRepository = userRepository;
    }

    /**
     * Obtener todas las sesiones activas del usuario actual
     */
    @GetMapping
    public List<SessionInfo> getMyActiveSessions(
            @AuthenticationPrincipal UserDetails userDetails) {
        UUID userId = getUserIdFromEmail(userDetails.getUsername());
        return authService.getActiveSessions(userId);
    }

    /**
     * Revocar una sesión específica (cerrar sesión desde un dispositivo)
     */
    @DeleteMapping("/{sessionId}")
    public void revokeSession(
            @PathVariable UUID sessionId,
            @AuthenticationPrincipal UserDetails userDetails) {
        UUID userId = getUserIdFromEmail(userDetails.getUsername());
        authService.revokeSession(sessionId, userId);
    }

    // Método auxiliar para obtener ID de usuario desde email
    private UUID getUserIdFromEmail(String email) {
        return userRepository.findByEmail(email)
            .orElseThrow(() -> new RuntimeException("User not found"))
            .getId();
    }
}
```

## Endpoint de Cierre de Sesión

El starter proporciona un endpoint de cierre de sesión en `/api/auth/logout`:

**Petición:**
```bash
POST /api/auth/logout
Content-Type: application/json
Authorization: Bearer {accessToken}

{
  "refreshToken": "tu-token-de-renovacion-aqui"
}
```

**Respuesta:**
```json
{
  "message": "Logged out successfully"
}
```

Este endpoint revoca el token de renovación proporcionado, evitando que se use para generar nuevos tokens de acceso.
