Todos los endpoints de autenticación tienen el prefijo definido en `ctlabs.auth.base-url` (predeterminado `/api/auth`).

## Registro e Inicio de Sesión

| Método | Endpoint | Descripción |
| :--- | :--- | :--- |
| `POST` | `/register` | Registra un usuario. Envía código de verificación si aplica. |
| `POST` | `/login` | Autentica y devuelve tokens de Acceso/Renovación. |
| `POST` | `/refresh-token` | Genera un nuevo Token de Acceso usando un Token de Renovación válido. |
| `POST` | `/logout` | Invalida el Token de Renovación especificado (cerrar sesión desde dispositivo actual). |

## Verificación

| Método | Endpoint | Descripción |
| :--- | :--- | :--- |
| `POST` | `/email-verification` | Verifica email usando el UUID/Token recibido. |
| `POST` | `/phone-verification` | Verifica teléfono usando el código OTP. |
| `POST` | `/resend-verification` | Reenvía código de verificación a email o número de teléfono. |

## Recuperación de Contraseña

| Método | Endpoint | Descripción |
| :--- | :--- | :--- |
| `POST` | `/forgot-password` | Inicia el flujo (envía código/enlace vía canal preferido). |
| `POST` | `/reset-password` | Establece la nueva contraseña usando el código/token. |

## Detalles de Endpoints

### POST `/register`

Registra un nuevo usuario en el sistema.

**Petición:**
```json
{
  "firstName": "John",
  "lastName": "Doe",
  "email": "john@example.com",
  "phoneNumber": "+1234567890",
  "password": "SecurePass123"
}
```

**Respuesta (200 OK):**
```json
{
  "message": "Registration successful. Please check your email to verify your account."
}
```

**Respuestas de Error:**
- `400 Bad Request` - El email ya existe
- `400 Bad Request` - Formato de email inválido
- `400 Bad Request` - La contraseña no cumple los requisitos de seguridad

---

### POST `/login`

Autentica un usuario y devuelve tokens de acceso y renovación.

**Petición:**
```json
{
  "username": "john@example.com",
  "password": "SecurePass123"
}
```

**Respuesta (200 OK):**
```json
{
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refreshToken": "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
}
```

::note
El campo `username` acepta tanto email como número de teléfono.
::

**Respuestas de Error:**
- `400 Bad Request` - Credenciales inválidas
- `400 Bad Request` - Email/Teléfono no verificado
- `400 Bad Request` - Cuenta deshabilitada

---

### POST `/refresh-token`

Genera un nuevo token de acceso usando un token de renovación válido.

**Petición:**
```json
{
  "refreshToken": "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
}
```

**Respuesta (200 OK):**
```json
{
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

**Respuestas de Error:**
- `400 Bad Request` - Token de renovación inválido
- `400 Bad Request` - Token de renovación expirado
- `400 Bad Request` - Token de renovación no encontrado

---

### POST `/email-verification`

Verifica la dirección de email de un usuario usando el código de verificación.

**Petición:**
```json
{
  "email": "john@example.com",
  "code": "abc123def456"
}
```

**Respuesta (200 OK):**
```json
{
  "message": "Email verified successfully. You can now log in."
}
```

**Respuestas de Error:**
- `400 Bad Request` - Código de verificación inválido o expirado
- `400 Bad Request` - Email ya verificado
- `400 Bad Request` - Usuario no encontrado

---

### POST `/phone-verification`

Verifica el número de teléfono de un usuario usando el código OTP.

**Petición:**
```json
{
  "phoneNumber": "+1234567890",
  "code": "123456"
}
```

::note
El número de teléfono debe estar en formato E.164 (ej., `+1234567890`).
::

**Respuesta (200 OK):**
```json
{
  "message": "Phone verified successfully. You can now log in."
}
```

**Respuestas de Error:**
- `400 Bad Request` - Código de verificación inválido o expirado
- `400 Bad Request` - Teléfono ya verificado
- `400 Bad Request` - Formato de número de teléfono inválido
- `400 Bad Request` - Usuario no encontrado

---

### POST `/forgot-password`

Inicia el flujo de restablecimiento de contraseña enviando un código de verificación.

**Petición:**
```json
{
  "username": "john@example.com"
}
```

::note
El campo `username` acepta tanto email como número de teléfono.
::

**Respuesta (200 OK):**
```json
{
  "message": "Password reset code sent to your email."
}
```

**Respuestas de Error:**
- `400 Bad Request` - Usuario no encontrado
- `400 Bad Request` - Email/Teléfono no verificado

---

### POST `/reset-password`

Restablece la contraseña del usuario usando el código de verificación.

**Petición:**
```json
{
  "username": "john@example.com",
  "code": "abc123def456",
  "newPassword": "NewSecurePass123"
}
```

**Respuesta (200 OK):**
```json
{
  "message": "Password reset successfully. You can now log in with your new password."
}
```

**Respuestas de Error:**
- `400 Bad Request` - Código de verificación inválido o expirado
- `400 Bad Request` - La contraseña no cumple los requisitos de seguridad
- `400 Bad Request` - Usuario no encontrado

---

### POST `/logout`

Invalida el token de renovación especificado, cerrando efectivamente la sesión del usuario desde el dispositivo actual.

**Petición:**
```json
{
  "refreshToken": "a1b2c3d4-e5f6-7890-abcd-ef1234567890:raw-token-here"
}
```

**Respuesta (200 OK):**
```json
{
  "message": "Logout successful."
}
```

**Respuestas de Error:**
- `400 Bad Request` - Formato de token de renovación inválido
- `400 Bad Request` - Token de renovación no encontrado
- `400 Bad Request` - Token de renovación inválido

::note
El endpoint de logout **no requiere autenticación** vía header Authorization. Solo valida el token de renovación en sí. Esto permite a los usuarios cerrar sesión incluso si su token de acceso ha expirado.
::

::tip
Para cerrar sesión desde todos los dispositivos, consulta la [Guía de Gestión de Sesiones](/guides/session-management).
::

### POST `/resend-verification`

Reenvía un código de verificación al email o número de teléfono del usuario. Elimina cualquier código de verificación existente y genera uno nuevo.

**Petición (Email):**
```json
{
  "username": "usuario@example.com",
  "channel": "email"
}
```

**Petición (Teléfono):**
```json
{
  "username": "+1234567890",
  "channel": "phone"
}
```

**Respuesta (200 OK) - Email:**
```json
{
  "message": "A new verification code has been sent to your email."
}
```

**Respuesta (200 OK) - Teléfono:**
```json
{
  "message": "A new verification code has been sent to your phone."
}
```

**Respuestas de Error:**
- `400 Bad Request` - Se requiere nombre de usuario
- `400 Bad Request` - El canal debe ser 'email' o 'phone'
- `400 Bad Request` - Usuario no encontrado
- `400 Bad Request` - El usuario no tiene email/teléfono
- `400 Bad Request` - Email/Teléfono ya está verificado
- `400 Bad Request` - La verificación de Email/Teléfono no está habilitada

::note
El campo `channel` define explícitamente si enviar vía email o SMS. El `username` puede ser el email o número de teléfono del usuario para identificarlo.
::

::tip
Usa el mismo identificador (email o teléfono) que se usó durante el registro para el campo `username`.
::

## Manejo de Errores

El starter usa un manejador global que devuelve respuestas estandarizadas:

```json
{
  "timestamp": "2024-01-01T12:00:00",
  "status": 400,
  "error": "Bad Request",
  "message": "Password does not meet security requirements",
  "path": "/api/auth/register"
}
```
