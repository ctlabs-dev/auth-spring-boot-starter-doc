## Instalación

Para agregar el starter a tu proyecto, inclúyelo como dependencia en tu archivo de compilación.

::code-group
```xml [pom.xml]
<dependency>
    <groupId>dev.ctlabs</groupId>
    <artifactId>auth-spring-boot-starter</artifactId>
    <version>0.1.0</version>
</dependency>
```
```groovy [build.gradle]
implementation 'dev.ctlabs:auth-spring-boot-starter:0.1.0'
```
::

## Configuración Mínima

Una vez instalado, agrega las siguientes propiedades obligatorias a tu `application.yml` o `application.properties`. Este es el mínimo requerido para que el sistema funcione.

::code-group
```properties [application.properties]
# Configuración Auth Starter
spring.datasource.url=jdbc:postgresql://localhost:5432/tu_bd
spring.datasource.username=tu_usuario
spring.datasource.password=tu_contraseña

ctlabs.auth.jwt.secret-key=${JWT_SECRET_KEY:clave_secreta_larga_y_segura}

# Excluir la configuración de seguridad predeterminada de Spring para compatibilidad
spring.autoconfigure.exclude=org.springframework.boot.security.autoconfigure.web.servlet.ServletWebSecurityAutoConfiguration
```
```yaml [application.yml]
spring:
  autoconfigure:
    exclude: org.springframework.boot.security.autoconfigure.web.servlet.ServletWebSecurityAutoConfiguration
  datasource:
    url: jdbc:postgresql://localhost:5432/tu_bd
    username: tu_usuario
    password: tu_contraseña

ctlabs:
  auth:
    jwt:
      secret-key: ${JWT_SECRET_KEY:clave_secreta_larga_y_segura}
```
::

::note
La `jwt.secret-key` es crítica para la seguridad. En entornos de producción, usa siempre variables de entorno o un gestor de secretos para protegerla.
::

## Gestión de Base de Datos

No necesitas crear ninguna tabla manualmente. **Auth Starter** usa Flyway para gestionar automáticamente el esquema de la base de datos. Tan pronto como inicies tu aplicación, las tablas necesarias para usuarios, roles y tokens se crearán en tu base de datos PostgreSQL.
