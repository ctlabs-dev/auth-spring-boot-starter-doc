Esta guía te mostrará la forma más rápida de ver Auth Starter en acción.

::tip
**¿Omitir la configuración manual?** Puedes descargar un proyecto de ejemplo listo para ejecutar desde el repositorio:

```shell
# Clonar el repositorio
git clone https://github.com/ctlabs-dev/auth-spring-boot-starter.git

# Navegar al ejemplo quickstart
cd auth-spring-boot-starter/examples/quickstart

# Ejecutar la aplicación
mvn spring-boot:run
```

El ejemplo está preconfigurado y listo para usar. Salta al [paso 5](#_5-probar-los-endpoints) para comenzar a probar los endpoints.
::

## 1. Crear un Proyecto Spring Boot

Usa [start.spring.io](https://start.spring.io/) para generar un nuevo proyecto Spring Boot con las siguientes dependencias:
- Spring Web
- Spring Data JPA
- PostgreSQL Driver

## 2. Agregar la Dependencia Auth Starter

Agrega la dependencia como se muestra en la guía de [Instalación](./installation).

## 3. Configurar [`application.properties`](./installation#minimum-configuration)

Configura el mínimo requerido como se describe en la guía de instalación, apuntando a tu base de datos PostgreSQL local.

## 4. Ejecutar la Aplicación

Inicia tu aplicación Spring Boot con `mvnw spring-boot:run`. Revisa los logs de la consola. Deberías ver la salida de Flyway indicando que las tablas `auth_` han sido creadas.

## 5. Probar los Endpoints

Los endpoints de autenticación ya están activos. Puedes usar una herramienta como `cURL` o Postman para interactuar con ellos.

::note{title="Modo Desarrollo"}
Por defecto, las cuentas se **verifican automáticamente** al registrarse (no se necesitan códigos de email/SMS). Esto es perfecto para pruebas, pero para producción querrás habilitar verificación real. Ver [Próximos Pasos](#proximos-pasos) abajo.
::

**Registrar un nuevo usuario:**
```bash
curl -X POST http://localhost:8080/api/auth/register \
-H "Content-Type: application/json" \
-d '{
  "email": "test@example.com",
  "password": "Password123!",
  "firstName": "Test",
  "lastName": "User"
}'
```

**Iniciar sesión con el usuario:**
```bash
curl -X POST http://localhost:8080/api/auth/login \
-H "Content-Type: application/json" \
-d '{
  "username": "test@example.com",
  "password": "Password123!"
}'
```

**Respuesta:**
```json
{
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refreshToken": "123:a1b2c3d4-e5f6-7890-abcd-ef1234567890",
}
```

¡Si fue exitoso, tienes un sistema de autenticación completamente funcional ejecutándose! Ahora puedes usar el `accessToken` en el header `Authorization: Bearer <token>` para acceder a endpoints protegidos.

**Cerrar sesión (opcional):**
```bash
curl -X POST http://localhost:8080/api/auth/logout \
-H "Content-Type: application/json" \
-d '{
  "refreshToken": "123:a1b2c3d4-e5f6-7890-abcd-ef1234567890"
}'
```

**Respuesta:**
```json
{
  "message": "Logout successful."
}
```
