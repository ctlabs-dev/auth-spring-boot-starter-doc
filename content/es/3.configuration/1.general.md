## URL Base

Configura la ruta base para todos los endpoints de autenticación.

| Propiedad | Descripción | Predeterminado |
| :--- | :--- | :--- |
| `ctlabs.auth.base-url` | El prefijo para todos los endpoints de autenticación. | `/api/auth` |

**Ejemplo:**

::code-group
```properties [application.properties]
ctlabs.auth.base-url=/api/auth
```

```yaml [application.yml]
ctlabs:
  auth:
    base-url: /api/auth
```
::

---

## Rutas Públicas

Configura qué endpoints deben ser accesibles sin autenticación. Se soportan patrones estilo Ant.

| Propiedad | Descripción | Predeterminado |
| :--- | :--- | :--- |
| `ctlabs.auth.public-paths` | Lista de rutas que no requieren autenticación. | `[]` (vacío) |

**Ejemplo:**

::code-group
```properties [application.properties]
ctlabs.auth.public-paths[0]=/api/public/**
ctlabs.auth.public-paths[1]=/webhook/**
ctlabs.auth.public-paths[2]=/health
```

```yaml [application.yml]
ctlabs:
  auth:
    public-paths:
      - /api/public/**
      - /webhook/**
      - /health
```
::

::note
Los endpoints de autenticación (`/api/auth/**`) son automáticamente públicos y no necesitan ser listados.
::

## JWT (Token de Acceso)

Configura la seguridad y vida útil de los tokens de acceso JWT.

| Propiedad | Descripción | Predeterminado |
| :--- | :--- | :--- |
| `ctlabs.auth.jwt.secret-key` | **Requerido**. Clave secreta usada para firmar JWTs. Debe ser de al menos 256 bits (32 bytes) para HS256. | `null` |
| `ctlabs.auth.jwt.expiration` | Vida útil del token de acceso. Soporta formato simple: `15m` para 15 minutos, `1h` para 1 hora, `24h` para 24 horas. | `15m` |

**Ejemplo:**

::code-group
```properties [application.properties]
ctlabs.auth.jwt.secret-key=tu-clave-secreta-de-al-menos-32-bytes-para-algoritmo-hs256
ctlabs.auth.jwt.expiration=15m
```

```yaml [application.yml]
ctlabs:
  auth:
    jwt:
      secret-key: tu-clave-secreta-de-al-menos-32-bytes-para-algoritmo-hs256
      expiration: 15m
```
::

::tip
**Generando una clave secreta segura:**
```bash
# Genera una cadena aleatoria de 256 bits (32 bytes)
openssl rand -base64 32
```
::

## Token de Renovación

Configura la vida útil de los tokens de renovación.

| Propiedad | Descripción | Predeterminado |
| :--- | :--- | :--- |
| `ctlabs.auth.refresh-token.expiration` | Vida útil del token de renovación. Soporta formato simple: `30d` para 30 días, `7d` para 7 días. | `30d` |

**Ejemplo:**

::code-group
```properties [application.properties]
ctlabs.auth.refresh-token.expiration=30d
```

```yaml [application.yml]
ctlabs:
  auth:
    refresh-token:
      expiration: 30d
```
::

## Rol Predeterminado

Configura qué rol se asigna automáticamente a nuevos usuarios al registrarse.

| Propiedad | Descripción | Predeterminado |
| :--- | :--- | :--- |
| `ctlabs.auth.default-role` | El nombre del rol asignado a nuevos usuarios. | `USER` |

**Ejemplo:**

::code-group
```properties [application.properties]
ctlabs.auth.default-role=USER
```

```yaml [application.yml]
ctlabs:
  auth:
    default-role: USER
```
::

::note
Si el rol no existe cuando un usuario se registra, el starter lo creará automáticamente en la base de datos.
::

## Ejemplo Completo

Una configuración mínima con todos los ajustes centrales:

::code-group
```properties [application.properties]
# URL Base
ctlabs.auth.base-url=/api/auth

# Rutas públicas
ctlabs.auth.public-paths[0]=/api/public/**
ctlabs.auth.public-paths[1]=/health

# JWT
ctlabs.auth.jwt.secret-key=tu-clave-secreta-de-al-menos-32-bytes-para-algoritmo-hs256
ctlabs.auth.jwt.expiration=15m

# Token de Renovación
ctlabs.auth.refresh-token.expiration=30d

# Rol Predeterminado
ctlabs.auth.default-role=USER
```

```yaml [application.yml]
ctlabs:
  auth:
    base-url: /api/auth
    public-paths:
      - /api/public/**
      - /health
    jwt:
      secret-key: tu-clave-secreta-de-al-menos-32-bytes-para-algoritmo-hs256
      expiration: 15m
    refresh-token:
      expiration: 30d
    default-role: USER
```
::
