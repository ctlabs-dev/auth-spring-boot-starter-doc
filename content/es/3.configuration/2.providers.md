## Resumen

El starter soporta múltiples proveedores de notificación para enviar códigos de verificación y enlaces de restablecimiento de contraseña. Puedes elegir entre email (SMTP, Brevo) y teléfono (SMS/WhatsApp vía Twilio o Brevo).

**Por defecto, ambos proveedores de email y teléfono están configurados en `NONE`**, lo que significa que no se envían mensajes y los usuarios son auto-verificados. Esto es útil para desarrollo pero debe cambiarse para producción.

---

## Proveedores de Email

El email se usa para:
- **Verificación de email** - Confirmar que el usuario posee la dirección de email
- **Restablecimiento de contraseña** - Enviar enlaces seguros de restablecimiento

### Proveedores Disponibles

| Proveedor | Descripción | Caso de Uso |
| :--- | :--- | :--- |
| `NONE` | No se envían emails. Los usuarios son auto-verificados. | Desarrollo, pruebas |
| `SMTP` | Email estándar vía cualquier servidor SMTP (Gmail, Outlook, etc.) | Auto-hospedado, configuración simple |
| `BREVO` | API de Brevo con plantillas transaccionales | Producción, emails con marca |

---

## Configuración SMTP

Usa cualquier servidor SMTP (Gmail, SendGrid, Amazon SES, etc.) para enviar emails.

### Propiedades

| Propiedad | Descripción | Predeterminado |
| :--- | :--- | :--- |
| `ctlabs.auth.notifications.mail.provider` | Proveedor de email. | `NONE` |
| `ctlabs.auth.notifications.mail.frontend-url` | URL del frontend para enlaces de verificación de email. | `http://localhost:3000` |
| `ctlabs.auth.notifications.mail.smtp.from-email` | Dirección de email desde la cual enviar. | `null` |
| `ctlabs.auth.notifications.mail.smtp.from-name` | Nombre de visualización para el remitente. | `null` |

### Ejemplo

::code-group
```properties [application.properties]
# Proveedor de Email
ctlabs.auth.notifications.mail.provider=SMTP
ctlabs.auth.notifications.mail.frontend-url=https://miapp.com
ctlabs.auth.notifications.mail.smtp.from-email=noreply@miapp.com
ctlabs.auth.notifications.mail.smtp.from-name=MiApp

# Spring Mail (SMTP) - Requerido para proveedor SMTP
spring.mail.host=smtp.gmail.com
spring.mail.port=587
spring.mail.username=tu-email@gmail.com
spring.mail.password=tu-contraseña-de-app
spring.mail.properties.mail.smtp.auth=true
spring.mail.properties.mail.smtp.starttls.enable=true
```

```yaml [application.yml]
ctlabs:
  auth:
    notifications:
      mail:
        provider: SMTP
        frontend-url: https://miapp.com
        smtp:
          from-email: noreply@miapp.com
          from-name: MiApp

# Spring Mail (SMTP) - Requerido para proveedor SMTP
spring:
  mail:
    host: smtp.gmail.com
    port: 587
    username: tu-email@gmail.com
    password: tu-contraseña-de-app
    properties:
      mail:
        smtp:
          auth: true
          starttls:
            enable: true
```
::

::tip
**Usuarios de Gmail:** Usa una [Contraseña de App](https://support.google.com/accounts/answer/185833) en lugar de tu contraseña regular. Habilita 2FA primero, luego genera una contraseña de app.
::

### Plantillas de Email (SMTP)

Al usar SMTP, el starter envía emails usando plantillas Thymeleaf integradas con las siguientes variables:

**Variables de plantilla disponibles:**
- `${name}` - Nombre del usuario
- `${confirmationUrl}` - URL completa de verificación o restablecimiento de contraseña
- `${expiration}` - Tiempo de expiración como número (ej. "24")
- `${unit}` - Etiqueta de unidad de tiempo (ej. "hours" o "minutes")
- `${currentYear}` - Año actual para pie de página de copyright

**Ejemplo de uso en plantillas:**
```html
<h1>Bienvenido, <span th:text="${name}">Usuario</span>!</h1>
<a th:href="${confirmationUrl}">Verificar Email</a>
<p>Este enlace expirará en <strong th:text="${expiration}">24</strong>
   <strong th:text="${unit}">horas</strong>.</p>
```

**Personalización de plantillas:**

Para personalizar las plantillas de email, crea tus propias plantillas Thymeleaf en tu proyecto:

```
src/main/resources/templates/
  ├── verification.html
  └── password-reset.html
```

Tus plantillas personalizadas sobrescribirán automáticamente las plantillas predeterminadas del starter. Consulta la [Guía de Personalización](/guides/customization) para ejemplos detallados.

## Configuración Brevo

Brevo (anteriormente Sendinblue) proporciona API de email transaccional con soporte de plantillas para emails profesionales con marca.

### Propiedades

| Propiedad | Descripción | Predeterminado |
| :--- | :--- | :--- |
| `ctlabs.auth.notifications.mail.provider` | Proveedor de email. | `NONE` |
| `ctlabs.auth.notifications.mail.frontend-url` | URL del frontend para enlaces de verificación de email. | `http://localhost:3000` |
| `ctlabs.auth.notifications.mail.brevo.api-key` | Clave API de Brevo (comienza con `xkeysib-`). | `null` |
| `ctlabs.auth.notifications.mail.brevo.base-url` | URL base de la API de Brevo. | `https://api.brevo.com/v3` |
| `ctlabs.auth.notifications.mail.brevo.verification-template-id` | ID de plantilla para verificación de email. | `null` |
| `ctlabs.auth.notifications.mail.brevo.password-reset-template-id` | ID de plantilla para restablecimiento de contraseña. | `null` |

### Ejemplo

::code-group
```properties [application.properties]
ctlabs.auth.notifications.mail.provider=BREVO
ctlabs.auth.notifications.mail.frontend-url=https://miapp.com
ctlabs.auth.notifications.mail.brevo.api-key=xkeysib-tu-clave-api-aqui
ctlabs.auth.notifications.mail.brevo.verification-template-id=1
ctlabs.auth.notifications.mail.brevo.password-reset-template-id=2
```

```yaml [application.yml]
ctlabs:
  auth:
    notifications:
      mail:
        provider: BREVO
        frontend-url: https://miapp.com
        brevo:
          api-key: xkeysib-tu-clave-api-aqui
          verification-template-id: 1
          password-reset-template-id: 2
```
::

::note
**Configuración de Plantillas:** Crea plantillas transaccionales en tu panel de Brevo. Usa las variables de plantilla (`{{params.name}}`, `{{params.url}}`, `{{params.expiration}}`) en el diseño de tu plantilla.
::

## Proveedores de Teléfono

Las notificaciones por teléfono se usan para:
- **Verificación por teléfono** - Enviar códigos OTP numéricos vía SMS o WhatsApp
- **Autenticación de dos factores** (si se implementa en tu app)

### Proveedores Disponibles

| Proveedor | Canales | Caso de Uso |
| :--- | :--- | :--- |
| `NONE` | Ninguno. Los usuarios son auto-verificados. | Desarrollo, apps solo con email |
| `TWILIO` | SMS, WhatsApp | Producción, cobertura global |
| `BREVO` | Solo SMS | Producción, enfoque europeo |

### Selección de Canal

| Canal | Descripción |
| :--- | :--- |
| `SMS` | Mensajes de texto estándar (predeterminado) |
| `WHATSAPP` | Mensajes de WhatsApp (solo Twilio) |

## Configuración Twilio

Twilio soporta tanto SMS como WhatsApp para enviar códigos de verificación.

### Propiedades

| Propiedad | Descripción | Predeterminado |
| :--- | :--- | :--- |
| `ctlabs.auth.notifications.phone.provider` | Proveedor de teléfono. | `NONE` |
| `ctlabs.auth.notifications.phone.channel` | Tipo de canal: `SMS` o `WHATSAPP`. | `SMS` |
| `ctlabs.auth.notifications.phone.from-phone-number` | Tu número de teléfono de Twilio. | `null` |
| `ctlabs.auth.notifications.phone.twilio.account-sid` | SID de Cuenta de Twilio. | `null` |
| `ctlabs.auth.notifications.phone.twilio.auth-token` | Token de Auth de Twilio. | `null` |
| `ctlabs.auth.notifications.phone.twilio.base-url` | URL base de la API de Twilio. | `https://api.twilio.com` |

### Ejemplo (SMS)

::code-group
```properties [application.properties]
ctlabs.auth.notifications.phone.provider=TWILIO
ctlabs.auth.notifications.phone.channel=SMS
ctlabs.auth.notifications.phone.from-phone-number=+1234567890
ctlabs.auth.notifications.phone.twilio.account-sid=tu-account-sid-de-twilio
ctlabs.auth.notifications.phone.twilio.auth-token=tu-auth-token-de-twilio
```

```yaml [application.yml]
ctlabs:
  auth:
    notifications:
      phone:
        provider: TWILIO
        channel: SMS
        from-phone-number: "+1234567890"
        twilio:
          account-sid: tu-account-sid-de-twilio
          auth-token: tu-auth-token-de-twilio
```
::

### Ejemplo (WhatsApp)

::code-group
```properties [application.properties]
ctlabs.auth.notifications.phone.provider=TWILIO
ctlabs.auth.notifications.phone.channel=WHATSAPP
ctlabs.auth.notifications.phone.from-phone-number=whatsapp:+1234567890
ctlabs.auth.notifications.phone.twilio.account-sid=tu-account-sid-de-twilio
ctlabs.auth.notifications.phone.twilio.auth-token=tu-auth-token-de-twilio
```

```yaml [application.yml]
ctlabs:
  auth:
    notifications:
      phone:
        provider: TWILIO
        channel: WHATSAPP
        from-phone-number: "whatsapp:+1234567890"
        twilio:
          account-sid: tu-account-sid-de-twilio
          auth-token: tu-auth-token-de-twilio
```
::

::tip
**Configuración de WhatsApp:** Necesitas un número habilitado para WhatsApp de Twilio. El `from-phone-number` debe tener el prefijo `whatsapp:` cuando uses el canal WhatsApp.
::

## Configuración de Teléfono Brevo

Brevo soporta SMS (no WhatsApp) para enviar códigos de verificación.

### Propiedades

| Propiedad | Descripción | Predeterminado |
| :--- | :--- | :--- |
| `ctlabs.auth.notifications.phone.provider` | Proveedor de teléfono. | `NONE` |
| `ctlabs.auth.notifications.phone.channel` | Tipo de canal (debe ser `SMS`). | `SMS` |
| `ctlabs.auth.notifications.phone.brevo.api-key` | Clave API de Brevo. | `null` |
| `ctlabs.auth.notifications.phone.brevo.base-url` | URL base de la API de Brevo. | `https://api.brevo.com/v3` |
| `ctlabs.auth.notifications.phone.brevo.sender-name` | Nombre del remitente SMS (máx. 11 caracteres alfanuméricos). | `AuthService` |

### Ejemplo

::code-group
```properties [application.properties]
ctlabs.auth.notifications.phone.provider=BREVO
ctlabs.auth.notifications.phone.channel=SMS
ctlabs.auth.notifications.phone.brevo.api-key=tu-clave-api-de-brevo
ctlabs.auth.notifications.phone.brevo.sender-name=MiApp
```

```yaml [application.yml]
ctlabs:
  auth:
    notifications:
      phone:
        provider: BREVO
        channel: SMS
        brevo:
          api-key: tu-clave-api-de-brevo
          sender-name: MiApp
```
::

::note
**Restricciones del Nombre de Remitente:** Los nombres de remitente SMS de Brevo deben tener 11 caracteres o menos y ser solo alfanuméricos (sin espacios ni caracteres especiales).
::

## Configuración de Verificación

Configura los tiempos de expiración y etiquetas de internacionalización para códigos de verificación y enlaces.

| Propiedad | Descripción | Predeterminado |
| :--- | :--- | :--- |
| `ctlabs.auth.verification.email-link-expiration` | Cuánto tiempo permanecen válidos los enlaces de verificación de email. | `24h` |
| `ctlabs.auth.verification.phone-code-expiration` | Cuánto tiempo permanecen válidos los códigos de verificación por teléfono. | `10m` |
| `ctlabs.auth.verification.unit-minutes` | Etiqueta para "minutos" en mensajes de verificación (para i18n). | `minutes` |
| `ctlabs.auth.verification.unit-hours` | Etiqueta para "horas" en mensajes de verificación (para i18n). | `hours` |

**Ejemplo:**

::code-group
```properties [application.properties]
ctlabs.auth.verification.email-link-expiration=24h
ctlabs.auth.verification.phone-code-expiration=10m
ctlabs.auth.verification.unit-minutes=minutos
ctlabs.auth.verification.unit-hours=horas
```

```yaml [application.yml]
# Con etiquetas en español para internacionalización
ctlabs:
  auth:
    verification:
      email-link-expiration: 24h
      phone-code-expiration: 10m
      unit-minutes: minutos
      unit-hours: horas
```
::

::note
Estas etiquetas se usan en los mensajes de verificación enviados a los usuarios. Ajústalas según el idioma de tu aplicación.
::
