All authentication endpoints have the prefix defined in `ctlabs.auth.base-url` (default `/api/auth`).

## Registration and Login

| Method | Endpoint | Description |
| :--- | :--- | :--- |
| `POST` | `/register` | Registers a user. Sends verification code if applicable. |
| `POST` | `/login` | Authenticates and returns Access/Refresh tokens. |
| `POST` | `/refresh-token` | Generates a new Access Token using a valid Refresh Token. |
| `POST` | `/logout` | Invalidates the specified Refresh Token (logout from current device). |

## Verification

| Method | Endpoint | Description |
| :--- | :--- | :--- |
| `POST` | `/email-verification` | Verifies email using the received UUID/Token. |
| `POST` | `/phone-verification` | Verifies phone using the OTP code. |
| `POST` | `/resend-verification` | Resends verification code to email or phone number. |

## Password Recovery

| Method | Endpoint | Description |
| :--- | :--- | :--- |
| `POST` | `/forgot-password` | Initiates the flow (sends code/link via preferred channel). |
| `POST` | `/reset-password` | Sets the new password using the code/token. |

## Endpoint Details

### POST `/register`

Registers a new user in the system.

**Request:**
```json
{
  "firstName": "John",
  "lastName": "Doe",
  "email": "john@example.com",
  "phoneNumber": "+1234567890",
  "password": "SecurePass123"
}
```

**Response (200 OK):**
```json
{
  "message": "Registration successful. Please check your email to verify your account."
}
```

**Error Responses:**
- `400 Bad Request` - Email already exists
- `400 Bad Request` - Invalid email format
- `400 Bad Request` - Password does not meet security requirements

---

### POST `/login`

Authenticates a user and returns access and refresh tokens.

**Request:**
```json
{
  "username": "john@example.com",
  "password": "SecurePass123"
}
```

**Response (200 OK):**
```json
{
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refreshToken": "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
}
```

::note
The `username` field accepts either email or phone number.
::

**Error Responses:**
- `400 Bad Request` - Invalid credentials
- `400 Bad Request` - Email/Phone not verified
- `400 Bad Request` - Account is disabled

---

### POST `/refresh-token`

Generates a new access token using a valid refresh token.

**Request:**
```json
{
  "refreshToken": "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
}
```

**Response (200 OK):**
```json
{
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

**Error Responses:**
- `400 Bad Request` - Invalid refresh token
- `400 Bad Request` - Refresh token expired
- `400 Bad Request` - Refresh token not found

---

### POST `/email-verification`

Verifies a user's email address using the verification code.

**Request:**
```json
{
  "email": "john@example.com",
  "code": "abc123def456"
}
```

**Response (200 OK):**
```json
{
  "message": "Email verified successfully. You can now log in."
}
```

**Error Responses:**
- `400 Bad Request` - Invalid or expired verification code
- `400 Bad Request` - Email already verified
- `400 Bad Request` - User not found

---

### POST `/phone-verification`

Verifies a user's phone number using the OTP code.

**Request:**
```json
{
  "phoneNumber": "+1234567890",
  "code": "123456"
}
```

::note
Phone number must be in E.164 format (e.g., `+1234567890`).
::

**Response (200 OK):**
```json
{
  "message": "Phone verified successfully. You can now log in."
}
```

**Error Responses:**
- `400 Bad Request` - Invalid or expired verification code
- `400 Bad Request` - Phone already verified
- `400 Bad Request` - Invalid phone number format
- `400 Bad Request` - User not found

---

### POST `/forgot-password`

Initiates the password reset flow by sending a verification code.

**Request:**
```json
{
  "username": "john@example.com"
}
```

::note
The `username` field accepts either email or phone number.
::

**Response (200 OK):**
```json
{
  "message": "Password reset code sent to your email."
}
```

**Error Responses:**
- `400 Bad Request` - User not found
- `400 Bad Request` - Email/Phone not verified

---

### POST `/reset-password`

Resets the user's password using the verification code.

**Request:**
```json
{
  "username": "john@example.com",
  "code": "abc123def456",
  "newPassword": "NewSecurePass123"
}
```

**Response (200 OK):**
```json
{
  "message": "Password reset successfully. You can now log in with your new password."
}
```

**Error Responses:**
- `400 Bad Request` - Invalid or expired verification code
- `400 Bad Request` - Password does not meet security requirements
- `400 Bad Request` - User not found

---

### POST `/logout`

Invalidates the specified refresh token, effectively logging out the user from the current device.

**Request:**
```json
{
  "refreshToken": "a1b2c3d4-e5f6-7890-abcd-ef1234567890:raw-token-here"
}
```

**Response (200 OK):**
```json
{
  "message": "Logout successful."
}
```

**Error Responses:**
- `400 Bad Request` - Invalid refresh token format
- `400 Bad Request` - Refresh token not found
- `400 Bad Request` - Invalid refresh token

::note
The logout endpoint **does not require authentication** via the Authorization header. It only validates the refresh token itself. This allows users to logout even if their access token has expired.
::

::tip
For logging out from all devices, see the [Session Management Guide](/guides/session-management).
::

### POST `/resend-verification`

Resends a verification code to the user's email or phone number. Deletes any existing verification code and generates a new one.

**Request (Email):**
```json
{
  "username": "user@example.com",
  "channel": "email"
}
```

**Request (Phone):**
```json
{
  "username": "+1234567890",
  "channel": "phone"
}
```

**Response (200 OK) - Email:**
```json
{
  "message": "A new verification code has been sent to your email."
}
```

**Response (200 OK) - Phone:**
```json
{
  "message": "A new verification code has been sent to your phone."
}
```

**Error Responses:**
- `400 Bad Request` - Username is required
- `400 Bad Request` - Channel must be 'email' or 'phone'
- `400 Bad Request` - User not found
- `400 Bad Request` - User does not have an email/phone
- `400 Bad Request` - Email/Phone is already verified
- `400 Bad Request` - Email/Phone verification is not enabled

::note
The `channel` field explicitly defines whether to send via email or SMS. The `username` can be the user's email or phone number to identify them.
::

::tip
Use the same identifier (email or phone) that was used during registration for the `username` field.
::

## Error Handling

The starter uses a global handler that returns standardized responses:

```json
{
  "timestamp": "2024-01-01T12:00:00",
  "status": 400,
  "error": "Bad Request",
  "message": "Password does not meet security requirements",
  "path": "/api/auth/register"
}
```
