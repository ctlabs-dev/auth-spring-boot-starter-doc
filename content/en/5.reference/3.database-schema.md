## Overview

The Auth Starter automatically creates and manages a set of database tables using Flyway migrations. All tables use the `auth_` prefix to avoid conflicts with your application's tables.

::note
This schema is created automatically when `ctlabs.auth.db.migration-enabled=true` (the default). See [Database Configuration](/configuration/database) for migration settings.
::

---

## Core Tables

### `auth_users`

Stores user credentials and verification status.

| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | `UUID` | Primary key (auto-generated) |
| `email` | `VARCHAR(255)` | User's email address (unique, nullable) |
| `phone_number` | `VARCHAR(20)` | User's phone number (unique, nullable) |
| `password_hash` | `VARCHAR(255)` | Bcrypt-hashed password |
| `is_email_verified` | `BOOLEAN` | Whether email has been verified (default: `false`) |
| `is_phone_verified` | `BOOLEAN` | Whether phone has been verified (default: `false`) |
| `status` | `VARCHAR(20)` | Account status: `active`, `suspended`, `deleted` (default: `active`) |
| `created_at` | `TIMESTAMP WITH TIME ZONE` | Account creation timestamp |
| `updated_at` | `TIMESTAMP WITH TIME ZONE` | Last update timestamp |

**Constraints:**
- Either `email` or `phone_number` must be provided (at least one non-null)
- Both `email` and `phone_number` are unique across the system

**Indexes:**
- `idx_auth_users_email` on `email`
- `idx_auth_users_phone` on `phone_number`

---

### `auth_profiles`

Stores user profile information (business data).

| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | `UUID` | Primary key (auto-generated) |
| `user_id` | `UUID` | Foreign key to `auth_users.id` (unique, cascade delete) |
| `first_name` | `VARCHAR(100)` | User's first name |
| `last_name` | `VARCHAR(100)` | User's last name |
| `avatar_url` | `TEXT` | URL to user's profile picture |
| `metadata` | `JSONB` | Additional custom profile data (flexible JSON storage) |
| `updated_at` | `TIMESTAMP WITH TIME ZONE` | Last update timestamp |

**Relationships:**
- One-to-one with `auth_users` (each user has exactly one profile)

---

### `auth_verification_codes`

Stores temporary verification codes for email/phone verification and password reset.

| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | `UUID` | Primary key (auto-generated) |
| `user_id` | `UUID` | Foreign key to `auth_users.id` (cascade delete) |
| `type` | `VARCHAR(50)` | Code type: `EMAIL_VERIFICATION`, `PHONE_VERIFICATION`, `PASSWORD_RESET` |
| `code` | `VARCHAR(255)` | The verification code (UUID for email, 6-digit OTP for phone) |
| `expires_at` | `TIMESTAMP WITH TIME ZONE` | Expiration timestamp |
| `created_at` | `TIMESTAMP WITH TIME ZONE` | Creation timestamp |

**Indexes:**
- `idx_auth_verification_code` on `code`

**Lifecycle:**
- Codes are automatically deleted after successful verification
- Expired codes should be cleaned up periodically

---

### `auth_refresh_tokens`

Stores active refresh tokens for session management.

| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | `UUID` | Primary key (auto-generated) |
| `user_id` | `UUID` | Foreign key to `auth_users.id` (cascade delete) |
| `token_hash` | `VARCHAR(255)` | Hashed refresh token (for security) |
| `device_info` | `TEXT` | Device information (user agent) |
| `ip_address` | `VARCHAR(45)` | IP address from which token was issued |
| `expires_at` | `TIMESTAMP WITH TIME ZONE` | Expiration timestamp |
| `revoked_at` | `TIMESTAMP WITH TIME ZONE` | Revocation timestamp (null if active) |
| `created_at` | `TIMESTAMP WITH TIME ZONE` | Creation timestamp |

**Usage:**
- One token per device/session
- Tokens are revoked on logout or password change
- Used to obtain new access tokens without re-login

## RBAC (Role-Based Access Control)

### `auth_roles`

Defines available roles in the system.

| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | `SERIAL` | Primary key (auto-increment) |
| `name` | `VARCHAR(50)` | Role name (unique, e.g., `ADMIN`, `EDITOR`, `VIEWER`) |
| `description` | `TEXT` | Human-readable description of the role |

**Pre-seeded Data:**
- `ADMIN` role is automatically created during migration
- Other roles are created dynamically when first assigned

---

### `auth_permissions`

Defines available permissions in the system.

| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | `SERIAL` | Primary key (auto-increment) |
| `slug` | `VARCHAR(50)` | Permission identifier (unique, e.g., `users:read`, `posts:write`) |
| `description` | `TEXT` | Human-readable description of the permission |

**Format:**
- Permissions use `resource:action` format (e.g., `users:read`, `posts:delete`)

---

### `auth_user_roles`

Many-to-many relationship between users and roles.

| Column | Type | Description |
| :--- | :--- | :--- |
| `user_id` | `UUID` | Foreign key to `auth_users.id` (cascade delete) |
| `role_id` | `INTEGER` | Foreign key to `auth_roles.id` (cascade delete) |

**Composite Primary Key:** `(user_id, role_id)`

---

### `auth_role_permissions`

Many-to-many relationship between roles and permissions.

| Column | Type | Description |
| :--- | :--- | :--- |
| `role_id` | `INTEGER` | Foreign key to `auth_roles.id` (cascade delete) |
| `permission_id` | `INTEGER` | Foreign key to `auth_permissions.id` (cascade delete) |

**Composite Primary Key:** `(role_id, permission_id)`

## Social Login (Optional)

### `auth_social_accounts`

Stores OAuth2 social login connections.

| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | `UUID` | Primary key (auto-generated) |
| `user_id` | `UUID` | Foreign key to `auth_users.id` (cascade delete) |
| `provider` | `VARCHAR(50)` | OAuth2 provider (e.g., `google`, `github`, `facebook`) |
| `provider_id` | `VARCHAR(255)` | User's ID in the provider's system |
| `created_at` | `TIMESTAMP WITH TIME ZONE` | Connection timestamp |

**Constraints:**
- Unique combination of `(provider, provider_id)`

**Indexes:**
- `idx_auth_social_lookup` on `(provider, provider_id)`

::note
Social login functionality is available but not yet fully documented. This table is created for future use.
::

## Audit & Security

### `auth_audit_logs`

Stores audit trail of authentication events.

| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | `BIGSERIAL` | Primary key (auto-increment) |
| `user_id` | `UUID` | Foreign key to `auth_users.id` (nullable, set null on delete) |
| `event_type` | `VARCHAR(50)` | Event type (e.g., `LOGIN`, `LOGOUT`, `PASSWORD_CHANGE`) |
| `ip_address` | `VARCHAR(45)` | IP address from which event occurred |
| `user_agent` | `TEXT` | Browser/client user agent string |
| `created_at` | `TIMESTAMP WITH TIME ZONE` | Event timestamp |

**Usage:**
- Tracks security-relevant events
- Used for security monitoring and compliance
- Can be queried for user activity history

## Migration History

### `ctlabs_auth_schema_history`

Flyway migration tracking table (created automatically by Flyway).

| Column | Type | Description |
| :--- | :--- | :--- |
| `installed_rank` | `INTEGER` | Order in which migrations were applied |
| `version` | `VARCHAR(50)` | Migration version number |
| `description` | `VARCHAR(200)` | Human-readable migration description |
| `type` | `VARCHAR(20)` | Migration type (e.g., `SQL`) |
| `script` | `VARCHAR(1000)` | Migration script filename |
| `checksum` | `INTEGER` | Migration script checksum |
| `installed_by` | `VARCHAR(100)` | Database user who applied the migration |
| `installed_on` | `TIMESTAMP` | When the migration was applied |
| `execution_time` | `INTEGER` | Execution time in milliseconds |
| `success` | `BOOLEAN` | Whether migration succeeded |

**Query Example:**
```sql
SELECT installed_rank, version, description, installed_on, success
FROM ctlabs_auth_schema_history
ORDER BY installed_rank;
```

## Schema Prefix

All tables are prefixed with `auth_` to avoid naming conflicts with your application's existing tables. This prefix is consistent across all migrations and cannot be customized.

::warning
Do not manually modify these tables unless you understand the implications. The starter expects this exact schema structure.
::
