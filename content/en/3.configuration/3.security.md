## Password Validation

Configure password requirements using a custom regex and validation message.

| Property | Description | Default |
| :--- | :--- | :--- |
| `ctlabs.auth.password.validation-regex` | Regex pattern for password validation. | `^(?=.*[0-9])(?=.*[a-z])(?=.*[A-Z])(?=\\S+$).{8,20}$` |
| `ctlabs.auth.password.validation-message` | Error message shown when password doesn't match the regex. | `Password must be 8-20 characters...` |

**Default requirements:**
- 8-20 characters long
- At least one digit
- At least one lowercase letter
- At least one uppercase letter
- No whitespace

**Example - Custom validation:**

::code-group
```properties [application.properties]
# Require 12+ chars, 1 digit, 1 special char
ctlabs.auth.password.validation-regex=^(?=.*[0-9])(?=.*[!@#$%^&*])(?=\\S+$).{12,}$
ctlabs.auth.password.validation-message=Password must be at least 12 characters with one number and one special character
```

```yaml [application.yml]
# Require 12+ chars, 1 digit, 1 special char
ctlabs:
  auth:
    password:
      validation-regex: "^(?=.*[0-9])(?=.*[!@#$%^&*])(?=\\S+$).{12,}$"
      validation-message: Password must be at least 12 characters with one number and one special character
```

## Admin User

Create an initial admin user automatically on first startup. This is useful for bootstrapping your application with an administrative account.

| Property | Description | Default |
| :--- | :--- | :--- |
| `ctlabs.auth.admin.enabled` | Whether to create admin user at startup. | `false` |
| `ctlabs.auth.admin.email` | Admin email address. | `null` |
| `ctlabs.auth.admin.password` | Admin password. | `null` |
| `ctlabs.auth.admin.first-name` | Admin first name. | `Admin` |
| `ctlabs.auth.admin.last-name` | Admin last name. | `System` |
| `ctlabs.auth.admin.role` | Admin role name. | `ADMIN` |
| `ctlabs.auth.admin.permissions` | List of permissions for the admin role. | `[]` (empty) |

**Example:**

::code-group
```properties [application.properties]
ctlabs.auth.admin.enabled=true
ctlabs.auth.admin.email=admin@myapp.com
ctlabs.auth.admin.password=AdminPassword123!
ctlabs.auth.admin.first-name=Admin
ctlabs.auth.admin.last-name=System
ctlabs.auth.admin.role=ADMIN
ctlabs.auth.admin.permissions[0]=users:write
ctlabs.auth.admin.permissions[1]=users:delete
ctlabs.auth.admin.permissions[2]=settings:manage
```

```yaml [application.yml]
ctlabs:
  auth:
    admin:
      enabled: true
      email: admin@myapp.com
      password: AdminPassword123!
      first-name: Admin
      last-name: System
      role: ADMIN
      permissions:
        - users:write
        - users:delete
        - settings:manage
```
::

::warning
**Security Best Practices:**
- Use environment variables for sensitive values
- Never commit passwords to version control
::

## API Key Authentication

Configure API Key authentication for trusted applications or mobile clients.

| Property | Description | Default |
| :--- | :--- | :--- |
| `ctlabs.auth.api-key-auth.enabled` | Whether API Key authentication is enabled. | `false` |
| `ctlabs.auth.api-key-auth.header-name` | The HTTP header name to look for the API Key. | `X-API-KEY` |
| `ctlabs.auth.api-key-auth.authorized-keys` | List of authorized API keys. | `[]` (empty) |

**Example:**

::code-group
```properties [application.properties]
ctlabs.auth.api-key-auth.enabled=true
ctlabs.auth.api-key-auth.header-name=X-API-KEY
ctlabs.auth.api-key-auth.authorized-keys[0]=my-secret-api-key-1
ctlabs.auth.api-key-auth.authorized-keys[1]=another-secret-key
```

```yaml [application.yml]
ctlabs:
  auth:
    api-key-auth:
      enabled: true
      header-name: X-API-KEY
      authorized-keys:
        - my-secret-api-key-1
        - another-secret-key
```

## Complete Example

::code-group
```properties [application.properties]
# Password Validation
ctlabs.auth.password.validation-regex=^(?=.*[0-9])(?=.*[a-z])(?=.*[A-Z])(?=\\S+$).{8,20}$
ctlabs.auth.password.validation-message=Password must be 8-20 characters long, contain at least one digit, one lowercase, one uppercase letter and no whitespace

# Admin User (disable after first run)
ctlabs.auth.admin.enabled=true
ctlabs.auth.admin.email=admin@myapp.com
ctlabs.auth.admin.password=AdminPassword123!
ctlabs.auth.admin.first-name=Admin
ctlabs.auth.admin.last-name=System
ctlabs.auth.admin.role=ADMIN
ctlabs.auth.admin.permissions[0]=users:write
ctlabs.auth.admin.permissions[1]=users:delete

# API Key Authentication
ctlabs.auth.api-key-auth.enabled=true
ctlabs.auth.api-key-auth.header-name=X-API-KEY
ctlabs.auth.api-key-auth.authorized-keys[0]=my-secret-api-key-1
ctlabs.auth.api-key-auth.authorized-keys[1]=another-secret-key
```

```yaml [application.yml]
ctlabs:
  auth:
    password:
      validation-regex: "^(?=.*[0-9])(?=.*[a-z])(?=.*[A-Z])(?=\\S+$).{8,20}$"
      validation-message: Password must be 8-20 characters long, contain at least one digit, one lowercase, one uppercase letter and no whitespace

    admin:
      enabled: true
      email: admin@myapp.com
      password: AdminPassword123!
      first-name: Admin
      last-name: System
      role: ADMIN
      permissions:
        - users:write
        - users:delete

    api-key-auth:
      enabled: true
      header-name: X-API-KEY
      authorized-keys:
        - my-secret-api-key-1
        - another-secret-key
```
