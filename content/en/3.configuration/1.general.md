## Base URL

Configure the base path for all authentication endpoints.

| Property | Description | Default |
| :--- | :--- | :--- |
| `ctlabs.auth.base-url` | The prefix for all authentication endpoints. | `/api/auth` |

**Example:**

::code-group
```properties [application.properties]
ctlabs.auth.base-url=/api/auth
```

```yaml [application.yml]
ctlabs:
  auth:
    base-url: /api/auth
```
::

---

## Public Paths

Configure which endpoints should be accessible without authentication. Ant-style patterns are supported.

| Property | Description | Default |
| :--- | :--- | :--- |
| `ctlabs.auth.public-paths` | List of paths that do not require authentication. | `[]` (empty) |

**Example:**

::code-group
```properties [application.properties]
ctlabs.auth.public-paths[0]=/api/public/**
ctlabs.auth.public-paths[1]=/webhook/**
ctlabs.auth.public-paths[2]=/health
```

```yaml [application.yml]
ctlabs:
  auth:
    public-paths:
      - /api/public/**
      - /webhook/**
      - /health
```
::

::note
Authentication endpoints (`/api/auth/**`) are automatically public and don't need to be listed.
::

## JWT (Access Token)

Configure the security and lifespan of JWT access tokens.

| Property | Description | Default |
| :--- | :--- | :--- |
| `ctlabs.auth.jwt.secret-key` | **Required**. Secret key used to sign JWTs. Must be at least 256 bits (32 bytes) for HS256. | `null` |
| `ctlabs.auth.jwt.expiration` | Access token lifespan. Supports simple format: `15m` for 15 minutes, `1h` for 1 hour, `24h` for 24 hours. | `15m` |

**Example:**

::code-group
```properties [application.properties]
ctlabs.auth.jwt.secret-key=your-secret-key-at-least-32-bytes-long-for-hs256-algorithm
ctlabs.auth.jwt.expiration=15m
```

```yaml [application.yml]
ctlabs:
  auth:
    jwt:
      secret-key: your-secret-key-at-least-32-bytes-long-for-hs256-algorithm
      expiration: 15m
```
::

::tip
**Generating a secure secret key:**
```bash
# Generate a 256-bit (32-byte) random string
openssl rand -base64 32
```
::

## Refresh Token

Configure the lifespan of refresh tokens.

| Property | Description | Default |
| :--- | :--- | :--- |
| `ctlabs.auth.refresh-token.expiration` | Refresh token lifespan. Supports simple format: `30d` for 30 days, `7d` for 7 days. | `30d` |

**Example:**

::code-group
```properties [application.properties]
ctlabs.auth.refresh-token.expiration=30d
```

```yaml [application.yml]
ctlabs:
  auth:
    refresh-token:
      expiration: 30d
```
::

## Default Role

Configure which role is automatically assigned to new users upon registration.

| Property | Description | Default |
| :--- | :--- | :--- |
| `ctlabs.auth.default-role` | The role name assigned to new users. | `USER` |

**Example:**

::code-group
```properties [application.properties]
ctlabs.auth.default-role=USER
```

```yaml [application.yml]
ctlabs:
  auth:
    default-role: USER
```
::

::note
If the role doesn't exist when a user registers, the starter will automatically create it in the database.
::

## Complete Example

A minimal configuration with all core settings:

::code-group
```properties [application.properties]
# Base URL
ctlabs.auth.base-url=/api/auth

# Public paths
ctlabs.auth.public-paths[0]=/api/public/**
ctlabs.auth.public-paths[1]=/health

# JWT
ctlabs.auth.jwt.secret-key=your-secret-key-at-least-32-bytes-long-for-hs256-algorithm
ctlabs.auth.jwt.expiration=15m

# Refresh Token
ctlabs.auth.refresh-token.expiration=30d

# Default Role
ctlabs.auth.default-role=USER
```

```yaml [application.yml]
ctlabs:
  auth:
    base-url: /api/auth
    public-paths:
      - /api/public/**
      - /health
    jwt:
      secret-key: your-secret-key-at-least-32-bytes-long-for-hs256-algorithm
      expiration: 15m
    refresh-token:
      expiration: 30d
    default-role: USER
```
::
