This guide walks you through setting up **real email verification** for user registrations. By the end, users will receive verification emails and must click a link to activate their accounts.

::note
The [Quickstart](/getting-started/quickstart) uses `Provider.NONE` for auto-verification. This guide shows how to enable **real email verification** with proof of ownership.
::

## Prerequisites

Choose one email provider:

### Option A: SMTP (Gmail, Outlook, etc.)
- A Gmail account (or other SMTP provider)
- App password enabled (for Gmail: [App Passwords Guide](https://support.google.com/accounts/answer/185833))

### Option B: Brevo (Recommended)
- Free [Brevo account](https://www.brevo.com/) (300 emails/day)
- API key from [Brevo Dashboard](https://app.brevo.com/settings/keys/api)
- Two email templates created (verification + password reset)

## Step 1: Configure Your Provider

### Using SMTP (Gmail Example)

::code-group
```properties [application.properties]
# Email Provider Configuration
ctlabs.auth.notifications.mail.provider=SMTP
ctlabs.auth.notifications.mail.frontend-url=http://localhost:3000

# SMTP Settings
ctlabs.auth.notifications.mail.smtp.from-email=your-email@gmail.com
ctlabs.auth.notifications.mail.smtp.from-name=Your App Name

# Spring Mail SMTP Configuration
spring.mail.host=smtp.gmail.com
spring.mail.port=587
spring.mail.username=your-email@gmail.com
spring.mail.password=your-app-password
spring.mail.properties.mail.smtp.auth=true
spring.mail.properties.mail.smtp.starttls.enable=true
```

```yaml [application.yml]
ctlabs:
  auth:
    notifications:
      mail:
        provider: SMTP
        frontend-url: http://localhost:3000
        smtp:
          from-email: your-email@gmail.com
          from-name: Your App Name

spring:
  mail:
    host: smtp.gmail.com
    port: 587
    username: your-email@gmail.com
    password: your-app-password
    properties:
      mail:
        smtp:
          auth: true
          starttls:
            enable: true
```
::

### Using Brevo

First, create your templates in Brevo:

1. Go to [Brevo Templates](https://app.brevo.com/camp/lists/template)
2. Create two templates:
   - **Email Verification Template** - Note the Template ID (e.g., `1`)
   - **Password Reset Template** - Note the Template ID (e.g., `2`)

**Required template variables:**
- `{{params.name}}` - User's first name
- `{{params.confirmationUrl}}` - Verification link (includes `code` and `email` query parameters) - **for email verification only**
- `{{params.resetUrl}}` - Password reset link (includes `code` and `email` query parameters) - **for password reset only**
- `{{params.expiration}}` - Expiration time (number)
- `{{params.unit}}` - Time unit (e.g., "hours")
- `{{params.currentYear}}` - Current year

::note
The verification URL will be in the format: `http://your-frontend.com/verify-email?code=abc123&email=user@example.com`

The password reset URL will be in the format: `http://your-frontend.com/reset-password?code=abc123&email=user@example.com`
::

::code-group
```properties [application.properties]
# Email Provider Configuration
ctlabs.auth.notifications.mail.provider=BREVO
ctlabs.auth.notifications.mail.frontend-url=http://localhost:3000

# Brevo Settings
ctlabs.auth.notifications.mail.brevo.api-key=xkeysib-your-api-key-here
ctlabs.auth.notifications.mail.brevo.verification-template-id=1
ctlabs.auth.notifications.mail.brevo.password-reset-template-id=2
```

```yaml [application.yml]
ctlabs:
  auth:
    notifications:
      mail:
        provider: BREVO
        frontend-url: http://localhost:3000
        brevo:
          api-key: xkeysib-your-api-key-here
          verification-template-id: 1
          password-reset-template-id: 2
```
::

## Step 2: Create Email Templates (SMTP Only)

If using SMTP, create Thymeleaf templates in your project:

**File:** `src/main/resources/templates/verification.html`

```html
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <title>Verify your email address</title>
</head>
<body>
    <h1>Welcome, <span th:text="${name}">User</span>!</h1>
    <p>Thanks for signing up. Please verify your email address:</p>
    <a th:href="${confirmationUrl}">Verify Email</a>
    <p>This link expires in <strong th:text="${expiration}">24</strong> <strong th:text="${unit}">hours</strong>.</p>
</body>
</html>
```

**File:** `src/main/resources/templates/password-reset.html`

```html
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <title>Reset your password</title>
</head>
<body>
    <h1>Password Reset Request</h1>
    <p>Hello, <span th:text="${name}">User</span>!</p>
    <p>Click the link below to reset your password:</p>
    <a th:href="${resetUrl}">Reset Password</a>
    <p>This link expires in <strong th:text="${expiration}">24</strong> <strong th:text="${unit}">hours</strong>.</p>
</body>
</html>
```

::note
Brevo users skip this step - templates are managed in the Brevo dashboard.

The `confirmationUrl` variable will contain the complete URL with `code` and `email` query parameters (e.g., `http://localhost:3000/verify-email?code=abc123&email=user@example.com`).
::

## Step 3: Test the Flow

### 1. Register a New User

```bash
curl -X POST http://localhost:8080/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "newuser@example.com",
    "password": "SecurePass123",
    "firstName": "Jane",
    "lastName": "Doe"
  }'
```

**Expected response:**
```json
{
  "message": "Registration successful. Please check your email to verify your account."
}
```

### 2. Check Your Email

- **SMTP:** Check the inbox of the registered email address
- **Brevo:** Check Brevo dashboard → Logs → Transactional to see delivery status

You should receive an email with a verification link like:
```
http://localhost:3000/verify-email?code=abc123xyz...&email=user@example.com
```

### 3. Verify the Account

The user clicks the link, which should call your backend:

```bash
curl -X POST http://localhost:8080/api/auth/verify-email \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "code": "abc123xyz..."
  }'
```

**Expected response:**
```json
{
  "message": "Email verified successfully. You can now log in."
}
```

### 3.1 Resend Code (Optional)

If the user doesn't receive the email or the code expires, they can request a new verification code:

```bash
curl -X POST http://localhost:8080/api/auth/resend-verification \
  -H "Content-Type: application/json" \
  -d '{
    "username": "newuser@example.com",
    "channel": "email"
  }'
```

**Expected response:**
```json
{
  "message": "A new verification code has been sent to your email."
}
```

::note
Requesting a new code will invalidate any previous verification codes for that user and channel.
::

### 4. Login

Now the user can log in:

```bash
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "newuser@example.com",
    "password": "SecurePass123"
  }'
```

**Expected response:**
```json
{
  "accessToken": "eyJhbGc...",
  "refreshToken": "5123e231-9bf7..."
}
```

## Troubleshooting

### Email Not Received (SMTP)

**Check spam folder** - Verification emails often end up in spam initially.

**Verify credentials:**
```bash
# Test SMTP connection
telnet smtp.gmail.com 587
```

**Enable "Less secure app access"** (Gmail):
- For Gmail, you **must** use an [App Password](https://support.google.com/accounts/answer/185833)
- Regular passwords won't work with SMTP

**Check logs:**
```bash
# Look for mail errors in your app logs
tail -f logs/application.log | grep -i "mail"
```

### Email Not Received (Brevo)

**Check Brevo logs:**
1. Go to [Brevo Transactional Logs](https://app.brevo.com/log/transactional)
2. Look for your email - status should be "Delivered"

**Common issues:**
- Invalid API key → Check `ctlabs.auth.notifications.mail.brevo.api-key`
- Template ID mismatch → Verify template IDs in configuration
- Email blocked → Check recipient is not in blocklist

### Token Expired

**Adjust expiration time:**

::code-group
```properties [application.properties]
# Increase to 48 hours
ctlabs.auth.verification.email-link-expiration=48h
```

```yaml [application.yml]
ctlabs:
  auth:
    verification:
      email-link-expiration: 48h
```
::

### Frontend URL Incorrect

The verification link uses `ctlabs.auth.notifications.mail.frontend-url` to build URLs:

```
{frontendUrl}/verify-email?code={code}&email={email}
```

Make sure this matches your frontend's actual URL:
- **Development:** `http://localhost:3000`
- **Production:** `https://yourapp.com`
