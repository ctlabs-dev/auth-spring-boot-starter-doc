::note

## Overview

The Auth Starter uses **refresh tokens** to manage sessions. Each time a user logs in:
- A new **access token** (JWT) is generated (expires in 15 minutes by default)
- A new **refresh token** is stored in the database (expires in 30 days by default)

**Available functionality:**
- ✅ View active sessions with details (device, IP, timestamps)
- ✅ Revoke a specific session (logout from one device)
- ✅ Logout endpoint provided by the starter

---

## Available Methods

The `AuthService` provides these methods for session management:

### 1. Get Active Sessions

```java
List<SessionInfo> getActiveSessions(UUID userId)
```

Returns detailed information about all active sessions for a user.

**SessionInfo includes:**
- `UUID id` - Session identifier
- `String deviceInfo` - Device/browser information
- `String ipAddress` - IP address of the session
- `Instant createdAt` - When session was created
- `Instant expiresAt` - When session expires

### 2. Revoke Session

```java
void revokeSession(UUID sessionId, UUID userId)
```

Revokes a specific session by its ID.

## Implementation Example

Create a controller to expose session management to authenticated users:

```java
package com.yourapp.controller;

import dev.ctlabs.starter.auth.application.dto.SessionInfo;
import dev.ctlabs.starter.auth.application.service.AuthService;
import dev.ctlabs.starter.auth.domain.repository.UserRepository;
import org.springframework.security.access.prepost.PreAuthorize;
import org.springframework.security.core.annotation.AuthenticationPrincipal;
import org.springframework.security.core.userdetails.UserDetails;
import org.springframework.web.bind.annotation.*;

import java.util.List;
import java.util.UUID;

@RestController
@RequestMapping("/api/me/sessions")
@PreAuthorize("isAuthenticated()")
public class UserSessionController {

    private final AuthService authService;
    private final UserRepository userRepository;

    public UserSessionController(AuthService authService, UserRepository userRepository) {
        this.authService = authService;
        this.userRepository = userRepository;
    }

    /**
     * Get all active sessions for the current user
     */
    @GetMapping
    public List<SessionInfo> getMyActiveSessions(
            @AuthenticationPrincipal UserDetails userDetails) {
        UUID userId = getUserIdFromEmail(userDetails.getUsername());
        return authService.getActiveSessions(userId);
    }

    /**
     * Revoke a specific session (logout from one device)
     */
    @DeleteMapping("/{sessionId}")
    public void revokeSession(
            @PathVariable UUID sessionId,
            @AuthenticationPrincipal UserDetails userDetails) {
        UUID userId = getUserIdFromEmail(userDetails.getUsername());
        authService.revokeSession(sessionId, userId);
    }

    // Helper method to get user ID from email
    private UUID getUserIdFromEmail(String email) {
        return userRepository.findByEmail(email)
            .orElseThrow(() -> new RuntimeException("User not found"))
            .getId();
    }
}
```

## Logout Endpoint

The starter provides a logout endpoint at `/api/auth/logout`:

**Request:**
```bash
POST /api/auth/logout
Content-Type: application/json
Authorization: Bearer {accessToken}

{
  "refreshToken": "your-refresh-token-here"
}
```

**Response:**
```json
{
  "message": "Logged out successfully"
}
```

This endpoint revokes the provided refresh token, preventing it from being used to generate new access tokens.
