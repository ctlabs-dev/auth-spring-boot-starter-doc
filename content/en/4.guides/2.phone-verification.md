This guide walks you through setting up **phone verification** for user registrations. Users will receive a verification code via SMS or WhatsApp and must enter it to activate their accounts.

::note
The [Quickstart](/getting-started/quickstart) uses `Provider.NONE` for auto-verification. This guide shows how to enable **real phone verification** with SMS or WhatsApp.
::

## Prerequisites

Choose one provider:

### Option A: Twilio (Most Popular)
- [Twilio account](https://www.twilio.com/try-twilio) (free trial with test credits)
- Phone number from [Twilio Console](https://console.twilio.com/us1/develop/phone-numbers/manage/incoming)
- Account SID and Auth Token from [Twilio Console](https://console.twilio.com/)
- For WhatsApp: [WhatsApp-enabled number](https://console.twilio.com/us1/develop/sms/try-it-out/whatsapp-learn)

### Option B: Brevo
- [Brevo account](https://www.brevo.com/) (SMS pricing varies by country)
- API key from [Brevo Dashboard](https://app.brevo.com/settings/keys/api)
- SMS credits purchased

## Step 1: Configure Your Provider

### Using Twilio

::code-group
```properties [application.properties]
# Phone Provider Configuration
ctlabs.auth.notifications.phone.provider=TWILIO
ctlabs.auth.notifications.phone.channel=SMS
ctlabs.auth.notifications.phone.from-phone-number=+1234567890

# Twilio Settings
ctlabs.auth.notifications.phone.twilio.account-sid=ACxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
ctlabs.auth.notifications.phone.twilio.auth-token=your_auth_token_here
```

```yaml [application.yml]
ctlabs:
  auth:
    notifications:
      phone:
        provider: TWILIO
        channel: SMS
        from-phone-number: +1234567890
        twilio:
          account-sid: ACxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
          auth-token: your_auth_token_here
```
::

**For WhatsApp**, change the channel and use WhatsApp format:

::code-group
```properties [application.properties]
ctlabs.auth.notifications.phone.channel=WHATSAPP
ctlabs.auth.notifications.phone.from-phone-number=whatsapp:+1234567890
```

```yaml [application.yml]
ctlabs:
  auth:
    notifications:
      phone:
        channel: WHATSAPP
        from-phone-number: whatsapp:+1234567890
```
::

### Using Brevo

::code-group
```properties [application.properties]
# Phone Provider Configuration
ctlabs.auth.notifications.phone.provider=BREVO
ctlabs.auth.notifications.phone.channel=SMS

# Brevo Settings
ctlabs.auth.notifications.phone.brevo.api-key=xkeysib-your-api-key-here
ctlabs.auth.notifications.phone.brevo.sender-name=YourApp
```

```yaml [application.yml]
ctlabs:
  auth:
    notifications:
      phone:
        provider: BREVO
        channel: SMS
        brevo:
          api-key: xkeysib-your-api-key-here
          sender-name: YourApp
```
::

::note
**Brevo only supports SMS**, not WhatsApp. The sender name must be max 11 alphanumeric characters.
::

## Step 2: Test the Flow

### 1. Register a New User with Phone

```bash
curl -X POST http://localhost:8080/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "phoneNumber": "+1234567890",
    "password": "SecurePass123",
    "firstName": "John",
    "lastName": "Doe"
  }'
```

**Expected response:**
```json
{
  "message": "Registration successful. A verification code has been sent to your phone."
}
```

### 2. Check Your Phone

You should receive an SMS or WhatsApp message with a 6-digit code:

```
Your verification code is: 123456
This code expires in 10 minutes.
```

### 3. Verify the Account

The user enters the code in your app, which calls:

```bash
curl -X POST http://localhost:8080/api/auth/verify-phone \
  -H "Content-Type: application/json" \
  -d '{
    "phoneNumber": "+1234567890",
    "code": "123456"
  }'
```

**Expected response:**
```json
{
  "message": "Phone verified successfully. You can now log in."
}
```

### 4. Login

Now the user can log in:

```bash
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "+1234567890",
    "password": "SecurePass123"
  }'
```

**Expected response:**
```json
{
  "accessToken": "eyJhbGc...",
  "refreshToken": "5123e231-9bf7..."
}
```

---

## Step 3: Resend Code (Optional)

If the user doesn't receive the code, they can request a new one:

```bash
curl -X POST http://localhost:8080/api/auth/resend-verification \
  -H "Content-Type: application/json" \
  -d '{
    "username": "+1234567890",
    "channel": "phone"
  }'
```

**Expected response:**
```json
{
  "message": "A new verification code has been sent to your phone."
}
```

## Troubleshooting

### SMS Not Received (Twilio)

**Check phone number format:**
- Must include country code: `+1234567890` (not `234567890`)
- No spaces or dashes

**Verify Twilio credentials:**
```bash
# Test Twilio API
curl -X POST "https://api.twilio.com/2010-04-01/Accounts/YOUR_ACCOUNT_SID/Messages.json" \
  --data-urlencode "Body=Test message" \
  --data-urlencode "From=+1234567890" \
  --data-urlencode "To=+1987654321" \
  -u YOUR_ACCOUNT_SID:YOUR_AUTH_TOKEN
```

**Check Twilio logs:**
1. Go to [Twilio Console](https://console.twilio.com/us1/monitor/logs/sms)
2. Look for your message - status should be "Delivered"

**Common issues:**
- **Unverified number** (trial account) → Add recipient to [Verified Caller IDs](https://console.twilio.com/us1/develop/phone-numbers/manage/verified)
- **Insufficient credits** → Add funds to your account
- **Country restrictions** → Some countries require registration

### SMS Not Received (Brevo)

**Check Brevo logs:**
1. Go to [Brevo SMS Logs](https://app.brevo.com/log/sms)
2. Look for your message

**Common issues:**
- Invalid API key → Check `ctlabs.auth.notifications.phone.brevo.api-key`
- Insufficient SMS credits → Purchase credits in Brevo dashboard
- Invalid sender name → Max 11 alphanumeric chars

### WhatsApp Not Working (Twilio)

**Enable WhatsApp:**
1. Go to [Twilio WhatsApp Sandbox](https://console.twilio.com/us1/develop/sms/try-it-out/whatsapp-learn)
2. Send the join code from your phone to activate
3. Use `whatsapp:+1234567890` format for the phone number

**Production WhatsApp:**
- Requires [Facebook Business Manager approval](https://www.twilio.com/docs/whatsapp/tutorial/connect-number-business-profile)
- Can take several days for approval

### Code Expired

**Adjust expiration time:**

::code-group
```properties [application.properties]
# Increase to 15 minutes
ctlabs.auth.verification.phone-code-expiration=15m
```

```yaml [application.yml]
ctlabs:
  auth:
    verification:
      phone-code-expiration: 15m
```
::

## Testing in Development

### Using Twilio Verify Test Numbers

For testing without sending real SMS:

**Test phone numbers (Twilio sandbox):**
- `+15005550006` - Always succeeds
- `+15005550001` - Always fails (invalid number)

**Update your test:**
```bash
curl -X POST http://localhost:8080/api/auth/register \
  -H "Content-Type: application/json" \

    "phoneNumber": "+15005550006",

```

```yaml [application-test.yml]
    notifications:
      phone:
        provider: NONE
```

## Cost Considerations

### Twilio Pricing (as of 2026)
- **SMS:** ~$0.0075 per message (varies by country)
- **WhatsApp:** ~$0.005-0.01 per message
- **Free trial:** $15 credit (enough for ~2000 SMS)

### Brevo Pricing
- **Pay-as-you-go:** Varies by country
- **Example:** ~$0.035 per SMS in US
- **No free tier** for SMS

### Optimization Tips
- **Use email for non-critical accounts** (free)
- **Use WhatsApp** when possible (cheaper than SMS with Twilio)
- **Monitor usage** to avoid unexpected costs
