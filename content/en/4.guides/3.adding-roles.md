This guide shows you how to create custom roles and permissions, assign them to users, and protect endpoints. By the end, you'll have a working role-based access control system.

::note
See [RBAC Concepts](/concepts/rbac) to understand roles vs permissions before following this guide.
::

- Completed the [Quickstart](/getting-started/quickstart)
- Basic understanding of Spring Security annotations

## Step 1: Create Roles and Permissions

The starter provides `UserManagementService` for managing roles and permissions. You can seed them at application startup or create them via API.

### Option A: Seed at Startup (Recommended)

Create a `CommandLineRunner` to initialize roles when your app starts:

```java
package com.yourapp.config;

import dev.ctlabs.starter.auth.application.service.UserManagementService;
import org.springframework.boot.CommandLineRunner;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class RoleSeeder {

    @Bean
    CommandLineRunner seedRoles(UserManagementService userManagementService) {
        return args -> {
            // Create roles
            userManagementService.createRole("ADMIN", "Administrator with full access");
            userManagementService.createRole("EDITOR", "Can create and edit content");
            userManagementService.createRole("VIEWER", "Can only view content");

            // Create permissions
            userManagementService.createPermission("users:read", "View users");
            userManagementService.createPermission("users:write", "Create/edit users");
            userManagementService.createPermission("users:delete", "Delete users");

            userManagementService.createPermission("posts:read", "View posts");
            userManagementService.createPermission("posts:write", "Create/edit posts");
            userManagementService.createPermission("posts:delete", "Delete posts");

            // Assign permissions to roles
            // ADMIN gets everything
            userManagementService.assignPermissionToRole("ADMIN", "users:read");
            userManagementService.assignPermissionToRole("ADMIN", "users:write");
            userManagementService.assignPermissionToRole("ADMIN", "users:delete");
            userManagementService.assignPermissionToRole("ADMIN", "posts:read");
            userManagementService.assignPermissionToRole("ADMIN", "posts:write");
            userManagementService.assignPermissionToRole("ADMIN", "posts:delete");

            // EDITOR can manage posts
            userManagementService.assignPermissionToRole("EDITOR", "posts:read");
            userManagementService.assignPermissionToRole("EDITOR", "posts:write");

            // VIEWER can only read
            userManagementService.assignPermissionToRole("VIEWER", "posts:read");
        };
    }
}
```

**Run your application:**
```bash
./mvnw spring-boot:run
```

The roles and permissions will be created automatically on startup.

### Option B: Create via API

If you prefer dynamic creation, expose an admin endpoint:

```java
package com.yourapp.controller;

import dev.ctlabs.starter.auth.application.service.UserManagementService;
import org.springframework.security.access.prepost.PreAuthorize;
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/api/admin/rbac")
@PreAuthorize("hasRole('ADMIN')")
public class RbacAdminController {

    private final UserManagementService userManagementService;

    public RbacAdminController(UserManagementService userManagementService) {
        this.userManagementService = userManagementService;
    }

    @PostMapping("/roles")
    public void createRole(@RequestBody CreateRoleRequest request) {
        userManagementService.createRole(request.name(), request.description());
    }

    @PostMapping("/permissions")
    public void createPermission(@RequestBody CreatePermissionRequest request) {
        userManagementService.createPermission(request.name(), request.description());
    }

    @PostMapping("/roles/{roleName}/permissions/{permissionName}")
    public void assignPermission(
            @PathVariable String roleName,
            @PathVariable String permissionName) {
        userManagementService.assignPermissionToRole(roleName, permissionName);
    }
}

record CreateRoleRequest(String name, String description) {}
record CreatePermissionRequest(String name, String description) {}
```

**Test the API:**
```bash
# Create a role
curl -X POST http://localhost:8080/api/admin/rbac/roles \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN" \
  -d '{
    "name": "MODERATOR",
    "description": "Can moderate content"
  }'

# Create a permission
curl -X POST http://localhost:8080/api/admin/rbac/permissions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN" \
  -d '{
    "name": "posts:moderate",
    "description": "Moderate user posts"
  }'

# Assign permission to role
curl -X POST http://localhost:8080/api/admin/rbac/roles/MODERATOR/permissions/posts:moderate \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN"
```

## Step 2: Assign Roles to Users

### Option A: Assign at Registration (Default Role)

Configure a default role for new users:

::code-group
```properties [application.properties]
# All new users get VIEWER role by default
ctlabs.auth.default-role=VIEWER
```

```yaml [application.yml]
ctlabs:
  auth:
    default-role: VIEWER
```
::

Now all registrations automatically get the VIEWER role.

### Option B: Assign Manually via API

Create an admin endpoint to assign roles:

```java
@RestController
@RequestMapping("/api/admin/users")
@PreAuthorize("hasRole('ADMIN')")
public class UserAdminController {

    private final UserManagementService userManagementService;

    public UserAdminController(UserManagementService userManagementService) {
        this.userManagementService = userManagementService;
    }

    @PostMapping("/{userId}/roles")
    public void assignRole(
            @PathVariable UUID userId,
            @RequestBody AssignRoleRequest request) {
        userManagementService.assignRole(userId, request.roleName());
    }

    @DeleteMapping("/{userId}/roles/{roleName}")
    public void removeRole(
            @PathVariable UUID userId,
            @PathVariable String roleName) {
        userManagementService.removeRole(userId, roleName);
    }
}

record AssignRoleRequest(String roleName) {}
```

**Test the API:**
```bash
# Assign EDITOR role to user
curl -X POST http://localhost:8080/api/admin/users/123e4567-e89b-12d3-a456-426614174000/roles \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN" \
  -d '{
    "roleName": "EDITOR"
  }'

# Remove VIEWER role from user
curl -X DELETE http://localhost:8080/api/admin/users/123e4567-e89b-12d3-a456-426614174000/roles/VIEWER \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN"
```

### Option C: Create Admin User at Startup

Configure an admin user in your properties:

::code-group
```properties [application.properties]
# Create admin user on startup
ctlabs.auth.admin.enabled=true
ctlabs.auth.admin.email=admin@yourapp.com
ctlabs.auth.admin.password=AdminSecure123!
ctlabs.auth.admin.first-name=System
ctlabs.auth.admin.last-name=Admin
ctlabs.auth.admin.role=ADMIN
ctlabs.auth.admin.permissions=users:read,users:write,users:delete,posts:read,posts:write,posts:delete
```

```yaml [application.yml]
ctlabs:
  auth:
    admin:
      enabled: true
      email: admin@yourapp.com
      password: AdminSecure123!
      first-name: System
      last-name: Admin
      role: ADMIN
      permissions:
        - users:read
        - users:write
        - users:delete
        - posts:read
        - posts:write
        - posts:delete
```
::

The admin user will be created automatically on first startup.

## Step 3: Protect Endpoints with Roles

Use Spring Security's `@PreAuthorize` annotation to protect your endpoints.

### Role-Based Protection

```java
package com.yourapp.controller;

import org.springframework.security.access.prepost.PreAuthorize;
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/api/posts")
public class PostController {

    // Anyone authenticated can read
    @GetMapping
    @PreAuthorize("isAuthenticated()")
    public List<Post> getAllPosts() {
        return postService.findAll();
    }

    // Only EDITOR or ADMIN can create
    @PostMapping
    @PreAuthorize("hasAnyRole('EDITOR', 'ADMIN')")
    public Post createPost(@RequestBody CreatePostRequest request) {
        return postService.create(request);
    }

    // Only ADMIN can delete
    @DeleteMapping("/{id}")
    @PreAuthorize("hasRole('ADMIN')")
    public void deletePost(@PathVariable Long id) {
        postService.delete(id);
    }
}
```

### Permission-Based Protection

For more granular control, use permissions:

```java
@RestController
@RequestMapping("/api/users")
public class UserController {

    // Requires specific permission
    @GetMapping
    @PreAuthorize("hasAuthority('users:read')")
    public List<User> getAllUsers() {
        return userService.findAll();
    }

    @PostMapping
    @PreAuthorize("hasAuthority('users:write')")
    public User createUser(@RequestBody CreateUserRequest request) {
        return userService.create(request);
    }

    @DeleteMapping("/{id}")
    @PreAuthorize("hasAuthority('users:delete')")
    public void deleteUser(@PathVariable UUID id) {
        userService.delete(id);
    }
}
```

### Class-Level Protection

Protect an entire controller:

```java
@RestController
@RequestMapping("/api/admin")
@PreAuthorize("hasRole('ADMIN')") // Applies to all methods
public class AdminController {

    @GetMapping("/dashboard")
    public DashboardData getDashboard() {
        return adminService.getDashboard();
    }

    @GetMapping("/stats")
    public Stats getStats() {
        return adminService.getStats();
    }

    // This method has additional permission requirement
    @DeleteMapping("/users/{id}")
    @PreAuthorize("hasAuthority('users:delete')")
    public void deleteUser(@PathVariable UUID id) {
        userService.delete(id);
    }
}
```

## Advanced: Complex Authorization Rules

### Combining Roles and Permissions

```java
// Must be ADMIN OR have specific permission
@PreAuthorize("hasRole('ADMIN') or hasAuthority('posts:moderate')")
@PostMapping("/{id}/approve")
public void approvePost(@PathVariable Long id) {
    postService.approve(id);
}
```

### Owner-Based Access

```java
// User can edit their own posts, or ADMIN can edit any
@PreAuthorize("hasRole('ADMIN') or @postSecurity.isOwner(#id, principal)")
@PutMapping("/{id}")
public Post updatePost(@PathVariable Long id, @RequestBody UpdatePostRequest request) {
    return postService.update(id, request);
}
```

**Create a security component:**
```java
@Component("postSecurity")
public class PostSecurityService {

    private final PostRepository postRepository;

    public PostSecurityService(PostRepository postRepository) {
        this.postRepository = postRepository;
    }

    public boolean isOwner(Long postId, Object principal) {
        if (!(principal instanceof UserDetails userDetails)) {
            return false;
        }

        Post post = postRepository.findById(postId).orElse(null);
        if (post == null) {
            return false;
        }

        return post.getAuthorEmail().equals(userDetails.getUsername());
    }
}
```

## Troubleshooting

### 403 Forbidden Error

**Check JWT contains roles:**
```bash
# Decode your token at https://jwt.io
# Look for "roles": ["ADMIN", "EDITOR"]
```

If roles are missing, the user may not have been assigned roles properly.

**Verify role exists:**
```sql
SELECT * FROM ctlabs_auth_roles WHERE name = 'ADMIN';
```

**Verify user has role:**
```sql
SELECT u.email, r.name
FROM ctlabs_auth_users u
JOIN ctlabs_auth_user_roles ur ON u.id = ur.user_id
JOIN ctlabs_auth_roles r ON ur.role_id = r.id
WHERE u.email = 'editor@test.com';
```

### Role Not Applied After Assignment

Roles are embedded in the JWT at login time. After assigning a new role, the user must **login again** to get a new token with updated roles.

```bash
# Assign role
curl -X POST http://localhost:8080/api/admin/users/USER_ID/roles \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -d '{"roleName":"EDITOR"}'

# User must login again to get new token
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"user@test.com","password":"Pass123!"}'
```

### Method Security Not Working

Ensure `@EnableMethodSecurity` is enabled (it's automatic with the starter):

```java
@Configuration
@EnableMethodSecurity // Should be present in SecurityConfig
public class SecurityConfig {
    // ...
}
```
