## Installation

To add the starter to your project, include it as a dependency in your build file.

::code-group
```xml [pom.xml]
<dependency>
    <groupId>dev.ctlabs</groupId>
    <artifactId>auth-spring-boot-starter</artifactId>
    <version>0.2.0</version>
</dependency>
```
```groovy [build.gradle]
implementation 'dev.ctlabs:auth-spring-boot-starter:0.2.0'
```
::

## Minimum Configuration

Once installed, add the following mandatory properties to your `application.yml` or `application.properties`. This is the bare minimum required to get the system running.
::code-group
```properties [application.properties]
# Auth Starter Configuration
spring.datasource.url=jdbc:postgresql://localhost:5432/your_db
spring.datasource.username=your_user
spring.datasource.password=your_password

ctlabs.auth.jwt.secret-key=${JWT_SECRET_KEY:long_and_secure_secret_key}
```
```yaml [application.yml]
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/your_db
    username: your_user
    password: your_password

ctlabs:
  auth:
    jwt:
      secret-key: ${JWT_SECRET_KEY:long_and_secure_secret_key}
```
::

::note
The `jwt.secret-key` is critical for security. In production environments, always use environment variables or a secrets manager to protect it.
::
## Database Management

You don't need to manually create any tables. **Auth Starter** uses Flyway to automatically manage the database schema. As soon as you start your application, the necessary tables for users, roles, and tokens will be created in your PostgreSQL database.
