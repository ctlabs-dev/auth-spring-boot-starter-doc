This guide will walk you through the fastest way to see the Auth Starter in action.

::tip
**Skip the manual setup?** You can download a ready-to-run example project from the repository:

```shell
# Clone the repository
git clone https://github.com/ctlabs-dev/auth-spring-boot-starter.git

# Navigate to the quickstart example
cd auth-spring-boot-starter/examples/quickstart

# Run the application
mvn spring-boot:run
```

The example is pre-configured and ready to use. Jump to [step 5](#_5-test-the-endpoints) to start testing the endpoints.
::

## 1. Create a Spring Boot Project

Use [start.spring.io](https://start.spring.io/) to generate a new Spring Boot project with the following dependencies:
- Spring Web
- Spring Data JPA
- PostgreSQL Driver

## 2. Add the Auth Starter Dependency

Add the dependency as shown in the [Installation](./installation) guide.

## 3. Configure [`application.properties`](./installation#minimum-configuration)

Set up the minimum configuration as described in the installation guide, pointing to your local PostgreSQL database.

## 4. Run the Application

Start your Spring Boot application `mvnw spring-boot:run`. Check the console logs. You should see output from Flyway indicating that the `auth_` tables have been created.

## 5. Test the Endpoints

The authentication endpoints are now live. You can use a tool like `cURL` or Postman to interact with them.

::note{title="Development Mode"}
By default, accounts are **automatically verified** upon registration (no email/SMS codes needed). This is perfect for testing, but for production, you'll want to enable real verification. See [Next Steps](#next-steps) below.
::

**Register a new user:**
```bash
curl -X POST http://localhost:8080/api/auth/register \
-H "Content-Type: application/json" \
-d '{
  "email": "test@example.com",
  "password": "Password123!",
  "firstName": "Test",
  "lastName": "User"
}'
```

**Login with the user:**
```bash
curl -X POST http://localhost:8080/api/auth/login \
-H "Content-Type: application/json" \
-d '{
  "username": "test@example.com",
  "password": "Password123!"
}'
```

**Response:**
```json
{
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refreshToken": "123:a1b2c3d4-e5f6-7890-abcd-ef1234567890",
}
```

If successful, you have a fully functional authentication system running! You can now use the `accessToken` in the `Authorization: Bearer <token>` header to access protected endpoints.

**Logout (optional):**
```bash
curl -X POST http://localhost:8080/api/auth/logout \
-H "Content-Type: application/json" \
-d '{
  "refreshToken": "123:a1b2c3d4-e5f6-7890-abcd-ef1234567890"
}'
```

**Response:**
```json
{
  "message": "Logout successful."
}
```
